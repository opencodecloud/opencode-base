# Yml 组件方案

## 1. 组件概述

`opencode-base-yml` 是一个轻量级 YAML 处理组件，基于 SPI 机制适配 SnakeYAML 等实现，提供统一的 YAML 读写 API。组件专注于配置文件场景，提供安全解析、路径访问、占位符解析、配置绑定与多文档合并能力。

核心特性：
- **统一门面 API**：`OpenYml.load()` / `OpenYml.dump()` / `OpenYml.bind()` 简洁易用
- **文档模型**：`YmlDocument` 封装解析结果，支持路径访问和类型转换
- **节点树**：`YmlNode` 接口提供 YAML 节点的类型安全遍历
- **配置绑定**：支持 YAML 到 Java Bean 的自动映射，使用注解自定义绑定
- **占位符解析**：支持 `${key}` 和 `${key:default}` 语法
- **多文档合并**：支持多策略 YAML 文档合并（覆盖、深合并等）
- **安全解析**：防止 YAML 炸弹、限制别名数量、嵌套深度和文档大小
- **SPI 可插拔**：通过 `YmlProvider` 接口支持不同的底层实现

## 3. 核心 API

### 3.1 OpenYml

> YAML 操作的主入口门面类，提供加载、转储、绑定、合并、占位符解析等静态方法。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `load(String yaml)` | 将 YAML 字符串解析为 `Map<String, Object>` |
| `parse(String yaml)` | 将 YAML 字符串解析为 `YmlDocument` |
| `loadFile(Path path)` | 从文件加载 YAML 文档 |
| `loadFile(Path path, Charset charset)` | 指定编码从文件加载 YAML 文档 |
| `loadStream(InputStream input)` | 从输入流加载 YAML 文档 |
| `loadAll(String yaml)` | 加载多文档 YAML（`---` 分隔） |
| `dump(Object data)` | 将对象转储为 YAML 字符串 |
| `dump(Object data, YmlConfig config)` | 使用指定配置转储为 YAML |
| `dump(YmlDocument document)` | 将 YmlDocument 转储为 YAML |
| `dumpObject(Object object)` | 将 Java 对象转储为 YAML |
| `writeFile(Object data, Path path)` | 将数据写入 YAML 文件 |
| `writeFile(Object data, Path path, Charset charset)` | 指定编码写入 YAML 文件 |
| `write(Object data, Writer writer)` | 将数据写入 Writer |
| `dumpAll(Iterable<?> documents)` | 将多个文档转储为 YAML |
| `bind(YmlDocument document, Class<T> type)` | 将文档绑定到 Java 对象 |
| `bind(YmlDocument document, String path, Class<T> type)` | 将文档子路径绑定到 Java 对象 |
| `bind(String yaml, Class<T> type)` | 将 YAML 字符串绑定到 Java 对象 |
| `toMap(Object object)` | 将 Java 对象转换为 Map |
| `merge(Map base, Map overlay)` | 合并两个 Map（默认策略） |
| `merge(Map base, Map overlay, MergeStrategy strategy)` | 使用指定策略合并 |
| `mergeAll(Map... maps)` | 合并多个 Map |
| `merge(YmlDocument base, YmlDocument overlay)` | 合并两个文档 |
| `resolvePlaceholders(String yaml)` | 解析 YAML 中的占位符 |
| `resolvePlaceholders(String yaml, PlaceholderResolver resolver)` | 使用自定义解析器解析占位符 |
| `resolvePlaceholders(Map data)` | 解析 Map 中的占位符 |
| `parseWithPlaceholders(String yaml)` | 解析 YAML 并解析占位符 |
| `parseWithPlaceholders(String yaml, Map properties)` | 使用自定义属性解析 |
| `isValid(String yaml)` | 验证 YAML 格式是否合法 |
| `parseTree(String yaml)` | 将 YAML 解析为节点树 |

**示例：**

```java
// 加载 YAML
Map<String, Object> data = OpenYml.load("server:\n  port: 8080");

// 解析为文档
YmlDocument doc = OpenYml.parse("server:\n  port: 8080");
String port = doc.getString("server.port"); // "8080"

// 从文件加载
YmlDocument doc = OpenYml.loadFile(Path.of("config.yml"));

// 转储为 YAML
String yaml = OpenYml.dump(data);

// 绑定到 Java Bean
ServerConfig config = OpenYml.bind(doc, "server", ServerConfig.class);

// 合并配置
Map<String, Object> merged = OpenYml.merge(base, overlay);

// 占位符解析
String resolved = OpenYml.resolvePlaceholders("port: ${SERVER_PORT:8080}");
```

### 3.2 YmlDocument

> 解析后的 YAML 文档封装，支持路径访问和类型转换。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `of(Object data)` | 从数据创建文档（静态工厂） |
| `empty()` | 创建空文档 |
| `getRoot()` | 获取根对象 |
| `asMap()` | 转换为 Map |
| `asList()` | 转换为 List |
| `get(String path)` | 通过路径获取值 |
| `get(String path, T defaultValue)` | 通过路径获取值，不存在返回默认值 |
| `getDocument(String path)` | 获取子文档 |
| `getString(String path)` | 获取字符串值 |
| `getString(String path, String defaultValue)` | 获取字符串值（带默认值） |
| `getInt(String path)` | 获取整数值 |
| `getInt(String path, int defaultValue)` | 获取整数值（带默认值） |
| `getLong(String path)` | 获取长整数值 |
| `getBoolean(String path)` | 获取布尔值 |
| `getBoolean(String path, boolean defaultValue)` | 获取布尔值（带默认值） |
| `getList(String path)` | 获取列表 |
| `getMap(String path)` | 获取 Map |
| `has(String path)` | 检查路径是否存在 |
| `getOptional(String path)` | 返回 Optional 包装值 |
| `keys()` | 获取所有顶层键 |
| `size()` | 获取元素数量 |
| `isEmpty()` | 检查是否为空 |

**示例：**

```java
YmlDocument doc = YmlDocument.of(data);

// 路径访问
String name = doc.getString("app.name");
int port = doc.getInt("server.port", 8080);
boolean debug = doc.getBoolean("debug", false);

// 子文档
YmlDocument server = doc.getDocument("server");

// Optional 访问
Optional<String> host = doc.getOptional("server.host");
```

### 3.3 YmlNode

> YAML 节点接口，表示 YAML 文档树中的一个节点（标量、序列或映射）。实现 `Iterable<YmlNode>`。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `getType()` | 获取节点类型（SCALAR / SEQUENCE / MAPPING） |
| `asText()` | 获取文本值 |
| `asText(String defaultValue)` | 获取文本值（带默认值） |
| `asInt()` / `asInt(int defaultValue)` | 获取整数值 |
| `asLong()` / `asLong(long defaultValue)` | 获取长整数值 |
| `asBoolean()` / `asBoolean(boolean defaultValue)` | 获取布尔值 |
| `asDouble()` / `asDouble(double defaultValue)` | 获取浮点值 |
| `get(String key)` | 获取映射中指定键的子节点 |
| `get(int index)` | 获取序列中指定索引的子节点 |
| `at(String path)` | 通过路径访问嵌套节点 |
| `has(String key)` | 检查是否包含指定键 |
| `size()` | 获取子节点数量 |
| `keys()` | 获取所有键 |
| `values()` | 获取所有子节点 |
| `toObject(Class<T> clazz)` | 转换为 Java 对象 |
| `toMap()` | 转换为 Map |
| `toList()` | 转换为 List |
| `toYaml()` | 转换为 YAML 字符串 |
| `getRawValue()` | 获取原始值 |

**示例：**

```java
YmlNode root = OpenYml.parseTree(yaml);
String city = root.at("address.city").asText();
int port = root.get("server").get("port").asInt();
```

**主要方法：**

|------|------|

### 3.5 YmlConfig

> YAML 处理的配置选项，通过 Builder 模式构建。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `defaults()` | 创建默认配置 |
| `builder()` | 创建配置构建器 |
| `getIndent()` | 获取缩进空格数 |
| `isPrettyPrint()` | 是否美化输出 |
| `isAllowDuplicateKeys()` | 是否允许重复键 |
| `isSafeMode()` | 是否启用安全模式 |
| `getMaxAliasesForCollections()` | 最大别名数量 |
| `getMaxNestingDepth()` | 最大嵌套深度 |
| `getMaxDocumentSize()` | 最大文档大小 |
| `getDefaultFlowStyle()` | 默认流式风格 |
| `getDefaultScalarStyle()` | 默认标量风格 |

**内部枚举：**
- `FlowStyle`：流式风格（BLOCK / FLOW / AUTO）
- `ScalarStyle`：标量风格（PLAIN / SINGLE_QUOTED / DOUBLE_QUOTED / LITERAL / FOLDED）

**Builder 方法：**

| 方法 | 描述 |
|------|------|
| `indent(int)` | 设置缩进空格数 |
| `prettyPrint(boolean)` | 设置美化输出 |
| `allowDuplicateKeys(boolean)` | 设置是否允许重复键 |
| `safeMode(boolean)` | 设置安全模式 |
| `maxAliasesForCollections(int)` | 设置最大别名数量 |
| `maxNestingDepth(int)` | 设置最大嵌套深度 |
| `maxDocumentSize(long)` | 设置最大文档大小 |
| `defaultFlowStyle(FlowStyle)` | 设置默认流式风格 |
| `defaultScalarStyle(ScalarStyle)` | 设置默认标量风格 |
| `build()` | 构建配置 |

**示例：**

```java
YmlConfig config = YmlConfig.builder()
    .indent(4)
    .prettyPrint(true)
    .safeMode(true)
    .maxNestingDepth(20)
    .build();

String yaml = OpenYml.dump(data, config);
```

### 3.6 ConfigBinder

> 配置绑定器，将 YAML 属性绑定到 Java 对象。支持从 YAML 字符串、PropertySource 或 Map 绑定。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `bind(String yamlContent, Class<T> clazz)` | 从 YAML 字符串绑定 |
| `bind(String yamlContent, String prefix, Class<T> clazz)` | 从 YAML 字符串的子路径绑定 |
| `bind(PropertySource source, Class<T> clazz)` | 从 PropertySource 绑定 |
| `bind(PropertySource source, String prefix, Class<T> clazz)` | 从 PropertySource 的子路径绑定 |
| `bind(Map properties, Class<T> clazz)` | 从 Map 绑定 |

**示例：**

```java
DatabaseConfig config = ConfigBinder.bind(yamlContent, DatabaseConfig.class);
ServerConfig server = ConfigBinder.bind(yamlContent, "server", ServerConfig.class);

PropertySource source = PropertySource.fromYaml(yamlContent);
AppConfig app = ConfigBinder.bind(source, AppConfig.class);
```

### 3.7 YmlBinder

> YAML 绑定器，将 YmlDocument 或原始数据绑定到 Java 对象。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `bind(YmlDocument document, Class<T> type)` | 从文档绑定 |
| `bind(YmlDocument document, String path, Class<T> type)` | 从文档子路径绑定 |
| `bind(Object data, Class<T> type)` | 从原始数据绑定 |
| `toMap(Object object)` | 将 Java 对象转换为 Map |

**示例：**

```java
ServerConfig config = YmlBinder.bind(document, ServerConfig.class);
ServerConfig config = YmlBinder.bind(document, "server", ServerConfig.class);
Map<String, Object> map = YmlBinder.toMap(config);
```

### 3.8 PropertySource

> 属性源接口，抽象配置属性的来源。提供 Map、YAML、Environment 三种内置实现。

**接口方法：**

| 方法 | 描述 |
|------|------|
| `getName()` | 获取属性源名称 |
| `getProperty(String path)` | 获取字符串属性 |
| `getProperty(String path, Class<T> type)` | 获取指定类型属性 |
| `containsProperty(String path)` | 检查属性是否存在 |
| `getPropertyNames()` | 获取所有属性名 |
| `getProperties()` | 获取所有属性 |
| `getProperties(String prefix)` | 获取指定前缀的属性 |

**内置实现：**
- `MapPropertySource`：基于 Map 的属性源
- `YamlPropertySource`：基于 YAML 字符串的属性源
- `EnvironmentPropertySource`：基于环境变量的属性源

**示例：**

```java
PropertySource source = PropertySource.fromYaml(yamlContent);
String host = source.getProperty("database.host");
int port = source.getProperty("database.port", Integer.class);
```

### 3.9 绑定注解

#### YmlProperty

> 将字段绑定到 YAML 属性。

```java
@YmlProperty("server.port")
private int port;
```

#### YmlValue

> 将 YAML 路径的值绑定到字段。

```java
@YmlValue("app.config.timeout")
private Duration timeout;
```

#### YmlAlias

> 定义替代属性名称。

```java
@YmlAlias({"host", "hostname", "server-host"})
private String host;
```

#### YmlIgnore

> 排除字段不参与 YAML 绑定。

```java
@YmlIgnore
private transient String internalState;
```

#### YmlNestedProperty

> 绑定嵌套对象。

```java
@YmlNestedProperty(prefix = "database")
private DatabaseConfig database;
```

### 3.11 YmlPath

> YAML 路径对象，表示文档中值的路径。支持属性段和索引段。使用 sealed 接口定义 Segment 类型。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `root()` | 创建根路径 |
| `of(String path)` | 从字符串解析路径 |
| `of(String... segments)` | 从多段创建路径 |
| `child(String property)` | 添加属性子路径 |
| `index(int index)` | 添加索引子路径 |
| `parent()` | 获取父路径 |
| `getSegments()` | 获取所有段 |
| `getLastSegment()` | 获取最后一段 |
| `isRoot()` | 是否为根路径 |
| `depth()` | 获取路径深度 |

**内部类型：**
- `Segment`：路径段的 sealed 接口
- `PropertySegment(String property)`：属性段 record
- `IndexSegment(int index)`：索引段 record

**示例：**

```java
YmlPath path = YmlPath.of("server.port");
YmlPath path = YmlPath.root().child("server").child("port");
YmlPath path = YmlPath.of("items[0].name");
```

### 3.12 PlaceholderResolver

> 占位符解析器，解析 `${...}` 占位符。支持自定义前缀/后缀、默认值分隔符、系统属性和环境变量。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `create()` | 创建默认解析器 |
| `create(Map<String, String> properties)` | 创建带属性的解析器 |
| `builder()` | 创建构建器 |
| `resolve(String text)` | 解析文本中的占位符 |
| `resolveYaml(String yaml)` | 解析 YAML 中的占位符 |
| `resolveMap(Map map)` | 解析 Map 中的占位符 |

**Builder 方法：**

| 方法 | 描述 |
|------|------|
| `prefix(String)` | 设置占位符前缀（默认 `${`） |
| `suffix(String)` | 设置占位符后缀（默认 `}`） |
| `defaultValueSeparator(String)` | 设置默认值分隔符（默认 `:`） |
| `strict(boolean)` | 是否严格模式（找不到属性时抛异常） |
| `addPropertySource(String name, Map source)` | 添加属性源 |
| `addResolver(Function<String, String>)` | 添加自定义解析函数 |
| `withSystemProperties()` | 包含系统属性 |
| `withEnvironmentVariables()` | 包含环境变量 |
| `build()` | 构建解析器 |

**示例：**

```java
PlaceholderResolver resolver = PlaceholderResolver.create()
    .builder()
    .withEnvironmentVariables()
    .build();

String resolved = resolver.resolve("Server: ${SERVER_HOST:localhost}");
```

### 3.13 PropertyPlaceholder

> 表示解析后的占位符，包含键和可选默认值。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `parse(String expression)` | 解析占位符表达式 |
| `parse(String expression, String prefix, String suffix, String separator)` | 使用自定义分隔符解析 |
| `of(String key)` | 创建占位符 |
| `of(String key, String defaultValue)` | 创建带默认值的占位符 |
| `getKey()` | 获取键 |
| `getDefaultValue()` | 获取默认值（可能为 null） |
| `defaultValue()` | 获取 Optional 默认值 |
| `hasDefaultValue()` | 是否有默认值 |
| `getRawExpression()` | 获取原始表达式 |

**示例：**

```java
PropertyPlaceholder ph = PropertyPlaceholder.parse("${server.port:8080}");
String key = ph.getKey();          // "server.port"
String def = ph.getDefaultValue(); // "8080"
```

### 3.24 异常体系

```
OpenYmlException (RuntimeException)
├── YmlParseException          // YAML 解析失败
├── YmlBindException           // 配置绑定失败
├── YmlPathException           // 路径访问失败
├── YmlPlaceholderException    // 占位符解析失败
└── YmlSecurityException       // 安全违规
```

#### OpenYmlException

> YAML 异常基类，包含可选的行号和列号信息。

| 方法 | 描述 |
|------|------|
| `OpenYmlException(String message)` | 创建异常 |
| `OpenYmlException(String message, int line, int column)` | 创建带位置的异常 |
| `OpenYmlException(String message, Throwable cause)` | 创建带原因的异常 |
| `getLine()` | 获取行号 |
| `getColumn()` | 获取列号 |
| `hasLocation()` | 是否有位置信息 |

#### YmlBindException

> 配置绑定异常，包含路径和目标类型信息。

| 方法 | 描述 |
|------|------|
| `getPath()` | 获取绑定路径 |
| `getTargetType()` | 获取目标类型 |

#### YmlPathException

> 路径访问异常，包含工厂方法。

| 方法 | 描述 |
|------|------|
| `getPath()` | 获取路径 |
| `indexOutOfBounds(String path, int index, int size)` | 索引越界异常（静态工厂） |
| `typeMismatch(String path, String expected, String actual)` | 类型不匹配异常（静态工厂） |

#### YmlPlaceholderException

> 占位符解析异常。

| 方法 | 描述 |
|------|------|
| `getPlaceholder()` | 获取占位符 |
| `circularReference(String placeholder)` | 循环引用异常（静态工厂） |

#### YmlSecurityException

> 安全异常，包含安全违规类型枚举。

**SecurityViolationType 枚举值：**
- `ALIAS_LIMIT_EXCEEDED`
- `NESTING_DEPTH_EXCEEDED`
- `DOCUMENT_SIZE_EXCEEDED`
- `FORBIDDEN_TYPE`
- `UNSAFE_TYPE`

| 方法 | 描述 |
|------|------|
| `getType()` | 获取违规类型 |
| `aliasLimitExceeded(int count, int limit)` | 别名超限（静态工厂） |
| `nestingDepthExceeded(int depth, int limit)` | 嵌套超限（静态工厂） |
| `documentSizeExceeded(long size, long limit)` | 文档超大（静态工厂） |
| `forbiddenType(String typeName)` | 禁止类型（静态工厂） |
| `unsafeType(String typeName)` | 不安全类型（静态工厂） |
