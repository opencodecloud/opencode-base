## 统一命名规范

所有 44 个组件遵循统一的类命名规范，确保 API 风格一致、易于记忆。

### 命名规则总览

| 类型 | 命名规范 | 说明 | 示例 |
|------|----------|------|------|
| **门面入口类** | `Open` + 功能名 | 用户主入口，提供静态 API | `OpenCache`, `OpenJson`, `OpenCrypto` |
| **工具类** | 功能名 + `Util` | 静态辅助方法集合 | `KeyUtil`, `CronUtil`, `TreeUtil` |
| **接口** | 无前缀 | 定义契约，语义清晰 | `Cache`, `Serializer`, `Validator` |
| **实现类** | 具体名称 + 类型后缀 | 算法/策略 + 类型 | `Argon2Hash`, `JacksonProvider` |
| **Record** | 无后缀 | 不可变数据实体 | `Email`, `Coordinate`, `DataPoint` |
| **枚举** | 无后缀 | 枚举类型 | `Currency`, `Priority` |
| **异常** | `Open` + 功能名 + `Exception` | 异常类 | `OpenCacheException`, `OpenValidationException` |
| **构建器** | 类名 + `Builder` | 构建器模式 | `EmailBuilder`, `ConfigBuilder` |

## 包结构规范

### 接口位置原则

**核心原则**：用户持有引用的接口放根包，内部扩展接口放子包

| 位置 | 放置内容 | 判断标准 |
|------|----------|----------|
| **根包** | 门面类、核心接口、常用 Record | 用户会作为变量类型、方法参数、返回类型持有 |
| **spi/** | SPI 扩展接口 | 用户实现以扩展功能，不直接持有 |
| **internal/** | 内部实现 | 用户不直接使用 |

### 标准包结构模板

```
cloud.opencode.base.{module}
├── Open{Module}.java          # 门面入口类
├── {CoreInterface}.java       # 核心接口（用户持有引用）
├── {CoreRecord}.java          # 核心 Record（用户持有引用）
├── Open{Module}Exception.java # 异常类
│
├── spi/                       # SPI 扩展接口
│   ├── {Provider}.java       # 用户实现的提供者接口
│   └── {Strategy}.java       # 用户实现的策略接口
│
├── internal/                  # 内部实现
│   └── Default{Impl}.java    # 默认实现类
│
└── {submodule}/               # 功能子模块
    └── ...
```

### JDK 25 特性使用规范

| 特性 | 使用场景 | 示例 |
|------|----------|------|
| **Record** | 不可变数据对象、DTO、元组 | `record Pair<L, R>(L left, R right) {}` |
| **Sealed Class** | 限定子类的类型层次 | `sealed interface Result permits Success, Failure {}` |
| **Pattern Matching for switch** | 类型安全的分支处理 | `switch (obj) { case String s -> ...; case Integer i -> ...; }` |
| **Pattern Matching for instanceof** | 类型检查与转换 | `if (obj instanceof String s) { use(s); }` |
| **Virtual Thread** | 高并发轻量级线程 | `Thread.startVirtualThread(() -> ...)` |
| **Sequenced Collections** | 有序集合操作 | `list.getFirst()`, `list.getLast()`, `list.reversed()` |
| **String Templates** (Preview) | 暂不使用 | 等待正式发布 |

```java
// Record 用于不可变数据
public record CacheEntry<V>(V value, Instant expireAt) {
    public boolean isExpired() {
        return Instant.now().isAfter(expireAt);
    }
}

// Sealed Class 用于限定类型层次
public sealed interface ValidationResult permits Valid, Invalid {
    boolean isValid();
}
public record Valid() implements ValidationResult {
    @Override public boolean isValid() { return true; }
}
public record Invalid(List<String> errors) implements ValidationResult {
    @Override public boolean isValid() { return false; }
}

// Pattern Matching for switch
public static String format(Object value) {
    return switch (value) {
        case null -> "null";
        case String s -> "\"" + s + "\"";
        case Number n -> n.toString();
        case Collection<?> c -> "[" + c.size() + " items]";
        default -> value.toString();
    };
}

// Virtual Thread 用于高并发场景
public static void processInParallel(List<Runnable> tasks) {
    try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
        tasks.forEach(executor::submit);
    }
}
```

### OpenException 基类设计

```java
package cloud.opencode.base.core.exception;

/**
 * OpenCode 组件统一异常基类
 * <p>
 * 所有组件异常都应继承此类，提供统一的异常处理能力
 */
public class OpenException extends RuntimeException {

    /** 错误码 */
    private final String errorCode;

    /** 组件名称 */
    private final String component;

    // ==================== 构造方法 ====================

    public OpenException(String message) {
        this(null, null, message, null);
    }

    public OpenException(String message, Throwable cause) {
        this(null, null, message, cause);
    }

    public OpenException(String component, String errorCode, String message) {
        this(component, errorCode, message, null);
    }

    public OpenException(String component, String errorCode, String message, Throwable cause) {
        super(message, cause);
        this.component = component;
        this.errorCode = errorCode;
    }

    // ==================== Getter ====================

    public String getErrorCode() {
        return errorCode;
    }

    public String getComponent() {
        return component;
    }

    // ==================== 格式化输出 ====================

    @Override
    public String getMessage() {
        StringBuilder sb = new StringBuilder();
        if (component != null) {
            sb.append("[").append(component).append("] ");
        }
        if (errorCode != null) {
            sb.append("(").append(errorCode).append(") ");
        }
        sb.append(super.getMessage());
        return sb.toString();
    }
}
```

### 组件异常模板

```java
package cloud.opencode.base.cache.exception;

import cloud.opencode.base.core.exception.OpenException;

/**
 * 缓存组件异常

public class OpenCacheException extends OpenException {

    private static final String COMPONENT = "Cache";

        super(COMPONENT, null, message);

    public OpenCacheException(String errorCode, String message) {
        super(COMPONENT, errorCode, message);
    }

    public OpenCacheException(String message, Throwable cause) {
        super(COMPONENT, null, message, cause);
    }

    public OpenCacheException(String errorCode, String message, Throwable cause) {
        super(COMPONENT, errorCode, message, cause);
    }

    // ==================== 静态工厂方法（常见异常） ====================

    public static OpenCacheException keyNotFound(String key) {
        return new OpenCacheException("CACHE_KEY_NOT_FOUND", "Cache key not found: " + key);
    }

    public static OpenCacheException loadFailed(String key, Throwable cause) {
        return new OpenCacheException("CACHE_LOAD_FAILED", "Failed to load cache value for key: " + key, cause);
    }

        return new OpenCacheException("CACHE_EXPIRED", "Cache entry expired: " + key);
    }
}
```

| 场景 | 使用异常 | 说明 |

```java
throw new IllegalArgumentException("参数不能为空");
throw new RuntimeException("操作失败", e);

// ✅ 正确示例 - 使用 OpenException 体系
throw new OpenIllegalArgumentException("参数不能为空");
throw new OpenException("操作失败", e);
```

```java
            // Then

        }
```

### 组件内部日志使用

```java
package cloud.opencode.base.cache;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class OpenCache {

    // 每个类一个 Logger，使用类名
    private static final Logger log = LoggerFactory.getLogger(OpenCache.class);

    public void put(String key, Object value) {
        // DEBUG: 详细操作信息
        log.debug("Putting cache entry: key={}, valueType={}", key, value.getClass().getSimpleName());

        try {
            // 业务逻辑
            doPut(key, value);

            // TRACE: 更详细的追踪
            log.trace("Cache entry stored successfully: key={}, hashCode={}", key, value.hashCode());

        } catch (Exception e) {
            // ERROR: 错误信息，包含异常
            log.error("Failed to put cache entry: key={}", key, e);
            throw new CacheException("Put failed", e);
        }
    }

    public static Cache<K, V> create(String name, Consumer<CacheConfig> configurer) {
        // INFO: 重要事件
        log.info("Creating cache: name={}", name);

        CacheConfig config = new CacheConfig();
        configurer.accept(config);

        // WARN: 潜在问题
        if (config.getMaxSize() <= 0) {
            log.warn("Cache maxSize not set, using default: {}", DEFAULT_MAX_SIZE);
            config.setMaxSize(DEFAULT_MAX_SIZE);
        }

        return new DefaultCache<>(name, config);
    }
}
```

### 方法设计规范

| 原则 | 说明 | 示例 |
|------|------|------|

## 推荐实现顺序

### 阶段一：核心基础
1. **001-Core** - 所有组件的基础

### 阶段二：高频基础工具
2. **112-String** - 字符串处理是最常用功能
3. **103-Collections** - 集合操作高频使用
4. **105-Date** - 日期时间处理必备
5. **111-Reflect** - 反射工具被多个组件依赖
6. **104-Crypto** - 加密功能安全相关
7. **101-Cache** - 缓存能力被广泛复用

### 阶段三：通用领域工具
8. **202-Json** - JSON处理几乎必需
9. **210-Validation** - 数据校验通用需求
10. **203-Log** - 日志功能基础设施
11. **207-Resilience** - 弹性组件提高稳定性
12. **206-Pool** - 对象池被多个组件使用

### 阶段四：按需实现
- 根据项目实际需求选择性实现其他组件
