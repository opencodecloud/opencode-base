# SMS 组件方案

## 1. 组件概述

### 1.1 设计目标

`opencode-base-sms` 模块提供统一的短信发送能力，支持多平台短信服务商对接。

**核心特性：**
- 多平台支持（阿里云、腾讯云、华为云、百度云、Twilio、AWS SNS 等）
- SPI 扩展机制，支持自定义服务商
- 模板管理与变量渲染
- 批量发送（支持并发、进度回调）
- 手机号验证与格式化
- 频率限制（分钟/小时/天三级窗口）
- 日志脱敏与敏感信息保护
- 完善的异常体系（网络异常、超时异常、限流异常、模板异常等）

## 3. 核心 API

### 3.1 OpenSms - 门面入口

`OpenSms` 是短信模块的主入口，采用实例模式（非静态），通过工厂方法创建。每个实例绑定一个 `SmsProvider` 和一个 `SmsTemplateRegistry`。

```java
public final class OpenSms {

    // === 工厂方法 ===

    /** 从配置创建 */
    public static OpenSms of(SmsConfig config)

    /** 从提供商创建 */
    public static OpenSms of(SmsProvider provider)

    /** 创建控制台模拟实例（测试用） */
    public static OpenSms console()

    // === 发送方法 ===

    /** 发送短信（手机号+内容） */
    public SmsResult send(String phoneNumber, String content)

    /** 发送短信消息对象 */
    public SmsResult send(SmsMessage message)

    /** 批量发送 */
    public List<SmsResult> sendBatch(List<SmsMessage> messages)

    /** 群发相同内容 */
    public List<SmsResult> sendToAll(List<String> phoneNumbers, String content)

    // === 模板方法 ===

    /** 注册模板（ID+内容） */
    public OpenSms registerTemplate(String id, String content)

    /** 注册模板对象 */
    public OpenSms registerTemplate(SmsTemplate template)

    /** 使用模板发送 */
    public SmsResult sendTemplate(String templateId, String phoneNumber,
                                  Map<String, String> variables)

    /** 使用模板群发 */
    public List<SmsResult> sendTemplateToAll(String templateId,
                                             List<String> phoneNumbers,
                                             Map<String, String> variables)

    // === 管理方法 ===

    /** 获取当前提供商 */
    public SmsProvider getProvider()

    /** 获取模板注册表 */
    public SmsTemplateRegistry getTemplateRegistry()

    /** 检查提供商是否可用 */
    public boolean isAvailable()

    /** 关闭提供商资源 */
    public void close()
}
```

**使用示例：**

```java
// 1. 快速开始 - 控制台模式（测试用）
OpenSms sms = OpenSms.console();
SmsResult result = sms.send("13800138000", "您的验证码是 123456");

// 2. 从配置创建
SmsConfig config = SmsConfig.builder()
    .providerType(SmsProviderType.ALIYUN)
    .accessKey("your-access-key")
    .secretKey("your-secret-key")
    .signName("OpenCode")
    .build();
OpenSms sms = OpenSms.of(config);
SmsResult result = sms.send("13800138000", "验证码：654321");

// 3. 使用模板发送
sms.registerTemplate("verify", "您的验证码是 ${code}，${expire}分钟内有效。");
SmsResult result = sms.sendTemplate("verify", "13800138000",
    Map.of("code", "123456", "expire", "5"));

// 4. 群发
List<SmsResult> results = sms.sendToAll(
    List.of("13800138001", "13800138002", "13800138003"),
    "系统维护通知：今晚22:00-次日06:00进行升级。");
```

### 3.2 SmsMessage - 短信消息

不可变 Record 类型，表示一条待发送的短信消息。

```java
public record SmsMessage(
    String phoneNumber,           // 手机号码
    String content,               // 消息内容（直接发送时使用）
    String templateId,            // 模板ID（模板发送时使用）
    Map<String, String> variables // 模板变量（不可变Map）
) {
    /** 使用内容创建消息 */
    public static SmsMessage of(String phoneNumber, String content)

    /** 使用模板创建消息 */
    public static SmsMessage ofTemplate(String phoneNumber, String templateId,
                                        Map<String, String> variables)

    /** 创建构建器 */
    public static Builder builder()
}
```

**Builder 方法：**

| 方法 | 参数 | 说明 |
|------|------|------|
| `phoneNumber(String)` | 手机号 | 设置接收手机号 |
| `content(String)` | 内容 | 设置短信正文 |
| `templateId(String)` | 模板ID | 设置模板标识 |
| `variables(Map)` | 变量Map | 批量设置模板变量 |
| `variable(String, String)` | key, value | 逐个设置模板变量 |
| `build()` | - | 构建 SmsMessage |

```java
// 直接发送
SmsMessage msg1 = SmsMessage.of("13800138000", "验证码：123456");

// 模板发送
SmsMessage msg2 = SmsMessage.ofTemplate("13800138000", "SMS_VERIFY",
    Map.of("code", "123456"));

// 构建器
SmsMessage msg3 = SmsMessage.builder()
    .phoneNumber("13800138000")
    .templateId("SMS_VERIFY")
    .variable("code", "123456")
    .variable("expire", "5")
    .build();
```

    /** 创建失败结果（含手机号） */
    public static SmsResult failure(String phoneNumber, String errorCode, String errorMessage)

### 3.4 SmsProvider - 提供商接口

所有短信服务商的统一接口，支持 SPI 扩展。

```java
public interface SmsProvider {

    /** 发送单条短信 */
    SmsResult send(SmsMessage message);

    /** 批量发送（默认逐条发送） */
    default List<SmsResult> sendBatch(List<SmsMessage> messages)

    /** 获取提供商名称 */
    String getName();

    /** 检查是否可用 */
    default boolean isAvailable()

    /** 关闭资源 */
    default void close()
}
```

**内置实现：**

| 实现类 | 说明 | 认证方式 |
|--------|------|----------|
| `AliSmsSender` | 阿里云短信 | HMAC-SHA1 签名 |
| `TencentSmsSender` | 腾讯云短信 | TC3-HMAC-SHA256 签名 |
| `HuaweiSmsSender` | 华为云短信 | WSSE 认证 |
| `HttpSmsProvider` | 通用 HTTP 提供商 | 自定义 |
| `ConsoleSmsProvider` | 控制台输出（测试） | 无 |

### 3.5 SmsProviderFactory - 提供商工厂

工厂类，支持通过配置创建提供商和 SPI 扩展。

```java
public final class SmsProviderFactory {

    /** 从配置创建提供商（自动识别类型） */
    public static SmsProvider create(SmsConfig config)

    /** 注册自定义提供商工厂 */
    public static void registerProvider(SmsProviderType type,
                                        Function<SmsConfig, SmsProvider> factory)

    /** 取消注册 */
    public static void unregisterProvider(SmsProviderType type)

    /** 获取控制台提供商 */
    public static SmsProvider console()

    /** SPI扩展接口 */
    public interface SpiProvider {
        boolean supports(SmsProviderType type);
        SmsProvider create(SmsConfig config);
    }
}
```

```java
// 注册自定义提供商
SmsProviderFactory.registerProvider(SmsProviderType.CUSTOM, config -> {
    return new MyCustomSmsProvider(config);
});

// 通过SPI扩展
// META-INF/services/cloud.opencode.base.sms.provider.SmsProviderFactory$SpiProvider
```

## 8. 线程安全与性能

### 8.1 线程安全保证

| 组件 | 线程安全机制 | 说明 |
|------|-------------|------|
| `OpenSms` | 不可变实例 | provider和registry在构造时确定 |
| `SmsMessage` | Record 不可变 | variables使用Map.copyOf |
| `SmsResult` | Record 不可变 | 构建后不可修改 |
| `SmsTemplate` | Record 不可变 | variableNames使用List.copyOf |
| `SmsTemplateRegistry` | `ConcurrentHashMap` | 模板注册并发安全 |
| `TemplateManager` | `ConcurrentHashMap` | 模板缓存并发安全 |
| `SmsRateLimiter` | `ConcurrentHashMap` + `AtomicInteger` | 计数器原子操作 |
| `SmsProviderFactory` | `ConcurrentHashMap` | 工厂注册并发安全 |
| `SecureSmsConfig` | `synchronized` + `volatile` | 清除与读取互斥 |
| `HttpSmsProvider` | `HttpClient` 线程安全 | JDK HttpClient 支持并发 |

### 8.2 性能基准

| 操作 | 吞吐量 | 延迟 (p99) | 说明 |
|------|-------|-----------|------|
| 单条发送 | ~100/s | <500ms | 受网络IO限制 |
| 批量发送 (100条) | ~500/s | <2s | BatchSender并行 |
| 模板渲染 | ~50,000/s | <1ms | 纯CPU计算 |
| 手机号验证 | ~100,000/s | <0.1ms | 正则匹配 |

## 9. FAQ

### Q1: OpenSms 是静态工具类还是实例？

`OpenSms` 是实例类，通过 `OpenSms.of()` 或 `OpenSms.console()` 创建。每个实例绑定一个提供商和一个模板注册表，支持在同一应用中使用多个提供商。

```java
OpenSms aliSms = OpenSms.of(aliConfig);
OpenSms tencentSms = OpenSms.of(tencentConfig);
OpenSms testSms = OpenSms.console();
```

### Q2: 如何实现自定义短信服务商？

```java
// 1. 实现 SmsProvider 接口
public class MySmsSender implements SmsProvider {
    @Override
    public SmsResult send(SmsMessage message) {
        // 调用自定义API...
        return SmsResult.success("MSG-123", message.phoneNumber());
    }

    @Override
    public String getName() { return "my-sms"; }
}

// 2. 通过 OpenSms.of() 使用
OpenSms sms = OpenSms.of(new MySmsSender());
sms.send("13800138000", "Hello");

// 3. 或通过工厂注册
SmsProviderFactory.registerProvider(SmsProviderType.CUSTOM,
    config -> new MySmsSender());
```

### Q3: 如何防止短信轰炸？

```java
SmsRateLimiter limiter = new SmsRateLimiter(1, 5, 10);
// 每分钟1条，每小时5条，每天10条

try {
    limiter.acquire("13800138000");
    sms.send("13800138000", "验证码：123456");
} catch (SmsRateLimitException e) {
    System.out.println("请 " + e.getRetryAfterSeconds() + " 秒后重试");
}
```

### Q4: 如何处理发送失败和重试？

```java
try {
    SmsResult result = sms.send(message);
    if (result.isFailed()) {
        // 业务级失败
        log.warn("发送失败: {}", result.errorMessage());
    }
} catch (SmsTimeoutException e) {
    // 超时可重试
    if (e.getTimeoutType() == SmsTimeoutException.TimeoutType.READ) {
        retryWithBackoff(() -> sms.send(message));
    }
} catch (SmsNetworkException e) {
    // 网络错误可重试
    retryWithBackoff(() -> sms.send(message));
} catch (SmsRateLimitException e) {
    // 限流，等待后重试
    Thread.sleep(e.getRetryAfterSeconds() * 1000);
    sms.send(message);
}
```

### Q5: 批量发送和单条发送有什么区别？

| 特性 | 单条发送 `send()` | 批量发送 `BatchSender` |
|------|-------------------|----------------------|
| API调用 | 每条1次 | 并发分批处理 |
| 并发控制 | 无 | 可配置并发数 |
| 进度回调 | 无 | 支持 `onProgress` |
| 手机号验证 | 不验证 | 可选验证 |
| 结果 | 单个 `SmsResult` | `BatchResult`（含统计） |
| 适用场景 | 实时发送（验证码） | 营销通知、批量通知 |
