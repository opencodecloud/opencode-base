## 架构总览

opencode-base 已完成瘦身，将 7 个可独立发展的领域模块拆分为 8 个独立项目（opencode-pro），保留 41 个纯基础工具模块。

```
独立项目（8 个，位于 opencode-pro/）
┌──────────────────────────────────────────────────────────────────┐
│  OpenData       数据库 ORM 框架        ← 原 base-db 完全抽走      │
│  OpenRes        容错框架               ← 原 base-resilience 完全抽走│
│  OpenValidation 校验框架               ← 原 base-validation 完全抽走│
│  OpenSecurity   安全框架               ← 原 base-security 抽走     │
│  OpenNet        网络框架               ← 原 base-net 完全抽走      │
│  OpenMetrics    可观测性框架            ← 原 base-observability 完全抽走│
│  OpenTasker     任务调度框架            ← 原 base-tasker 完全抽走   │
│  OpenCache      缓存框架               ← 继承扩展 base-cache       │
└──────────────────────────────────────────────────────────────────┘
         │ 依赖（compile scope）
         ▼
opencode-base（基础工具库，41 个模块）
┌─────────────────────────────────────────────────────────────────┐
│                      业务组件 (3xx) - 15 个                      │
│  Captcha Email Event Feature Geo Graph Image Lunar Money        │
│  SMS Test TimeSeries Tree Web Pdf                               │
├─────────────────────────────────────────────────────────────────┤
│                      领域组件 (2xx) - 10 个                      │
│  Config Functional Json Log Parallel Pool                       │
│  Serialization Xml Yml OAuth2                                   │
├─────────────────────────────────────────────────────────────────┤
│                      基础组件 (1xx) - 15 个                      │
│  Cache ClassLoader Collections Crypto Date DeepClone Hash I18n  │
│  ID IO Reflect String Expression Lock Rules                     │
├─────────────────────────────────────────────────────────────────┤
│                      核心组件 (0xx) - 1 个                       │
│                          Core                                    │
└─────────────────────────────────────────────────────────────────┘
```

所有组件的门面入口类均以 `Open` 开头（如 `OpenJson`、`OpenCrypto`），在 IDE 中输入 `Open` 即可自动补全找到所有入口。

## 核心基础

### Core — 核心工具库

**编号**: 001 | **文档**: [001-Core组件方案.md](001-Core组件方案.md)

所有组件的基础依赖，提供最常用的工具类和统一异常体系。

| 功能分类 | 门面类 | 典型用法 |
|---------|--------|---------|
| 对象操作 | `OpenObject` | 空判断、默认值、类型转换 |
| 数组操作 | `OpenArray` | 合并、查找、转换 |
| 数学运算 | `OpenMath` | 阶乘、GCD、素数、求和 |
| 进制转换 | `OpenHex` / `OpenBase64` | 十六进制、Base64 编解码 |
| 类型转换 | `Convert` | 万能类型转换器 |
| 元组 | `Pair` / `Triple` | 不可变二元组、三元组 |
| 前置条件 | `Preconditions` | 参数校验（Guava 风格）|
| 构建器 | `OpenBuilder` | 通用 Builder 模式 |

```java
// 空安全默认值
String name = OpenObject.defaultIfNull(user.getName(), "unknown");

// 通用构建器
User user = OpenBuilder.of(User::new)
    .with(User::setName, "Alice")
    .with(User::setAge, 25)
    .build();
```

**适用场景**: 每个项目都会用到，是所有组件的基础。

### Json 处理

**编号**: 202 | **文档**: [202-Json组件方案.md](202-Json组件方案.md)

JsonNode node = OpenJson.parse(json);

**适用场景**: 需要 JSON 序列化/反序列化、配置文件解析、API 数据处理。

String value = doc.xpath("//server/port").text();

**适用场景**: XML 配置解析、SOAP 报文处理、文档生成与验证。

## 安全与加密

### 加密解密

**编号**: 104 | **文档**: [104-Crypto组件方案.md](104-Crypto组件方案.md)

全面的加密工具集：对称加密 (AES-GCM/ChaCha20/SM4)、非对称加密 (RSA/SM2)、数字签名 (Ed25519/ECDSA)、哈希摘要、密码哈希 (Argon2/BCrypt)、PGP、JWT。

```java
// AES-GCM 加密
byte[] encrypted = OpenCrypto.aesGcm().setKey(key).encrypt("机密数据");

// 密码哈希
String hash = OpenCrypto.argon2().hash("password123");
boolean ok = OpenCrypto.argon2().verify("password123", hash);

// 数字签名
byte[] sig = OpenSign.ed25519().setPrivateKey(privKey).sign(data);
```

**适用场景**: 数据加密、密码存储、数字签名、JWT 令牌、PGP 加密邮件。

> **扩展**: 企业级安全能力（HSM、Vault 集成、证书自动管理、安全审计）请使用独立项目 [OpenSecurity](#opensecurity--安全框架)。

### OAuth2 认证

**编号**: 213 | **文档**: [213-OAuth2组件方案.md](213-OAuth2组件方案.md)

OAuth2/OIDC 客户端：支持 Google、GitHub、Microsoft 等主流平台，自动刷新令牌、PKCE。

```java
OAuth2Client client = OpenOAuth2.github()
    .clientId("xxx").clientSecret("yyy")
    .redirectUri("http://localhost/callback")
    .build();
String authUrl = client.getAuthorizationUrl();
```

**适用场景**: 第三方登录（微信/GitHub/Google）、OAuth2 授权流程。

### Hash 哈希

**编号**: 107 | **文档**: [107-Hash组件方案.md](107-Hash组件方案.md)

非加密哈希：MurmurHash3、xxHash、CRC32、一致性哈希、布隆过滤器。

```java
long hash = OpenHash.murmur3().hashString("data").asLong();
BloomFilter<String> filter = OpenHash.bloomFilter(10000, 0.01);
```

**适用场景**: 数据分片、去重、布隆过滤器、一致性哈希路由。

## 数据与存储

### 缓存

**编号**: 101 | **文档**: [101-Cache组件方案.md](101-Cache组件方案.md)

高性能本地缓存：LRU/LFU/W-TinyLfu 淘汰策略、TTL 过期、异步加载、多层缓存、写入穿透/回写、事件监听、指标统计。

## Maven 依赖引入

### opencode-base 模块

每个组件对应一个 Maven 模块，按需引入：

```xml
<!-- 引入单个组件 -->
<dependency>
    <groupId>cloud.opencode.base</groupId>
    <artifactId>opencode-base-json</artifactId>
    <version>${opencode.version}</version>
</dependency>

<!-- 常用组合：Core + Json + Cache -->
<dependency>
    <groupId>cloud.opencode.base</groupId>
    <artifactId>opencode-base-core</artifactId>
    <version>${opencode.version}</version>
</dependency>
<dependency>
    <groupId>cloud.opencode.base</groupId>
    <artifactId>opencode-base-json</artifactId>
    <version>${opencode.version}</version>
</dependency>
<dependency>
    <groupId>cloud.opencode.base</groupId>
    <artifactId>opencode-base-cache</artifactId>
    <version>${opencode.version}</version>
</dependency>
```

> **提示**: 所有组件都自动依赖 `opencode-base-core`，无需额外引入。

### 独立项目模块

```xml
<!-- 引入独立项目组件 -->
<dependency>
    <groupId>cloud.opencode.opendata</groupId>
    <artifactId>opencode-opendata-core</artifactId>
    <version>${opendata.version}</version>
</dependency>

<dependency>
    <groupId>cloud.opencode.openres</groupId>
    <artifactId>opencode-openres-core</artifactId>
    <version>${openres.version}</version>
</dependency>
```

### 统一 BOM 管理

推荐使用 BOM 统一管理版本，无需逐个指定版本号：

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>cloud.opencode</groupId>
            <artifactId>opencode-bom</artifactId>
            <version>2026.02</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<!-- 无需指定版本号 -->
<dependency>
    <groupId>cloud.opencode.base</groupId>
    <artifactId>opencode-base-json</artifactId>
</dependency>
<dependency>
    <groupId>cloud.opencode.opendata</groupId>
    <artifactId>opencode-opendata-core</artifactId>
</dependency>
```
