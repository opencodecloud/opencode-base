# I18n 组件方案

## 1. 组件概述

`opencode-base-i18n` 是一个高性能、可扩展的国际化消息组件，基于 SPI 架构实现消息提供、Locale 解析和消息格式化三大核心扩展点。支持 ResourceBundle、链式回退、缓存、热重载等多种消息源，提供位置参数、命名参数和模板表达式三种格式化方式，基于 JDK 25 实现，零外部依赖。

**适用场景：**
- Web 应用多语言文本翻译
- 错误码与消息的国际化映射
- 命名参数 `${name}` 和模板表达式格式化
- 消息文件热重载，无需重启

**模块依赖：**
```xml
<dependency>
    <groupId>cloud.opencode.base</groupId>
    <artifactId>opencode-base-i18n</artifactId>
    <version>${version}</version>
</dependency>
```

## 3. 核心 API

### 3.1 OpenI18n

> 国际化门面入口类，提供消息获取、Locale 管理和组件配置的统一入口。内部自动组装 `MessageProvider`、`LocaleResolver`、`MessageFormatter` 三大 SPI 组件。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `static String get(String key, Object... args)` | 获取当前 Locale 的消息，支持位置参数 |
| `static String get(String key, Locale locale, Object... args)` | 获取指定 Locale 的消息 |
| `static String get(String key, Map<String, Object> params)` | 获取消息，使用命名参数 |
| `static String get(String key, Locale locale, Map<String, Object> params)` | 获取指定 Locale 的消息，使用命名参数 |
| `static String getOrDefault(String key, String defaultValue, Object... args)` | 获取消息，不存在时返回默认值 |
| `static String getOrDefault(String key, Locale locale, String defaultValue, Object... args)` | 获取指定 Locale 消息，不存在时返回默认值 |
| `static boolean contains(String key)` | 判断当前 Locale 是否包含消息 |
| `static boolean contains(String key, Locale locale)` | 判断指定 Locale 是否包含消息 |
| `static Locale getCurrentLocale()` | 获取当前线程 Locale |
| `static void setCurrentLocale(Locale locale)` | 设置当前线程 Locale |
| `static void resetLocale()` | 重置当前线程 Locale |
| `static void withLocale(Locale locale, Runnable runnable)` | 在指定 Locale 下执行任务 |
| `static <T> T withLocale(Locale locale, Supplier<T> supplier)` | 在指定 Locale 下执行有返回值任务 |
| `static void setLocaleResolver(LocaleResolver resolver)` | 设置 Locale 解析器 |
| `static void setMessageProvider(MessageProvider provider)` | 设置消息提供者 |
| `static void setMessageFormatter(MessageFormatter formatter)` | 设置消息格式化器 |
| `static void setDefaultLocale(Locale locale)` | 设置默认 Locale |
| `static void setThrowOnMissingMessage(boolean throwOnMissing)` | 设置消息缺失时是否抛异常 |
| `static MessageSource getMessageSource()` | 获取内部 MessageSource |
| `static void refresh()` | 刷新所有消息缓存 |

**示例：**

```java
// 位置参数
String msg = OpenI18n.get("user.welcome", "张三");

// 命名参数
String msg2 = OpenI18n.get("user.greeting", Map.of("name", "Alice", "count", 5));

// 指定 Locale
String enMsg = OpenI18n.get("user.welcome", Locale.ENGLISH, "John");

// 安全获取
String safe = OpenI18n.getOrDefault("unknown.key", "默认消息");

// 临时切换 Locale
OpenI18n.withLocale(Locale.JAPANESE, () -> {
    String jpMsg = OpenI18n.get("user.welcome", "太郎");
});

// 配置组件
OpenI18n.setMessageProvider(new ResourceBundleProvider("i18n/messages"));
OpenI18n.setLocaleResolver(new ThreadLocalLocaleResolver(Locale.CHINESE));
```

### 3.2 MessageSource

> 消息源接口，定义消息检索的核心契约。`OpenI18n` 内部持有此接口的实现。

**主要方法：**

| 方法 | 描述 |

| `boolean containsMessage(String key, Locale locale)` | 是否包含消息 |
| `Set<String> getKeys(Locale locale)` | 获取指定 Locale 所有 key |
| `Set<Locale> getSupportedLocales()` | 获取支持的 Locale 集合 |

### 3.3 LocaleContext

> Locale 上下文记录类（record），封装地区和时区信息。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `Locale locale()` | 获取地区 |
| `TimeZone timeZone()` | 获取时区 |
| `static LocaleContext of(Locale locale)` | 创建只含地区的上下文 |
| `static LocaleContext of(Locale locale, TimeZone timeZone)` | 创建含地区和时区的上下文 |
| `static LocaleContext getDefault()` | 获取默认上下文 |
| `static LocaleContext withDefaultTimeZone(Locale locale)` | 创建使用默认时区的上下文 |

**示例：**

```java
LocaleContext ctx = LocaleContext.of(Locale.CHINESE);
LocaleContext ctxWithTz = LocaleContext.of(Locale.US, TimeZone.getTimeZone("America/New_York"));
Locale locale = ctx.locale();
TimeZone tz = ctxWithTz.timeZone();
```

### 3.9 ResourceBundleProvider

> 基于 Java ResourceBundle 的消息提供者，支持多 baseName 和自定义 ClassLoader。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `ResourceBundleProvider(String baseName)` | 创建单 baseName 提供者 |
| `ResourceBundleProvider(List<String> baseNames)` | 创建多 baseName 提供者 |
| `ResourceBundleProvider(String baseName, ClassLoader classLoader)` | 指定 ClassLoader |
| `ResourceBundleProvider(List<String> baseNames, ClassLoader classLoader)` | 指定 ClassLoader + 多 baseName |
| `Optional<String> getMessageTemplate(String key, Locale locale)` | 获取消息模板 |
| `boolean containsMessage(String key, Locale locale)` | 是否包含消息 |
| `Set<String> getKeys(Locale locale)` | 获取所有 key |
| `Set<Locale> getSupportedLocales()` | 获取支持的 Locale |
| `void refresh()` | 清除 ResourceBundle 缓存 |
| `void setUseCache(boolean useCache)` | 设置是否使用缓存 |
| `void setDefaultEncoding(String encoding)` | 设置文件编码 |
| `void addBaseName(String baseName)` | 添加 baseName |

**示例：**

```java

Optional<String> msg = provider.getMessageTemplate("user.welcome", Locale.CHINESE);

ResourceBundleProvider multi = new ResourceBundleProvider(
    List.of("i18n/messages", "i18n/errors", "i18n/validation"));
```

### 3.11 CachingMessageProvider

> 缓存装饰器，为任意 `MessageProvider` 添加本地缓存和 TTL 过期。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `CachingMessageProvider(MessageProvider delegate)` | 使用默认缓存配置 |
| `CachingMessageProvider(MessageProvider delegate, int cacheSize, Duration ttl)` | 指定缓存大小和 TTL |
| `Optional<String> getMessageTemplate(String key, Locale locale)` | 缓存获取消息 |
| `boolean containsMessage(String key, Locale locale)` | 委托查询 |
| `Set<String> getKeys(Locale locale)` | 委托查询 |
| `Set<Locale> getSupportedLocales()` | 委托查询 |
| `void refresh()` | 刷新并清除缓存 |
| `double getHitRate()` | 获取缓存命中率 |
| `int getCacheSize()` | 获取当前缓存条目数 |
| `void clearCache()` | 清除缓存 |

**示例：**

```java
MessageProvider delegate = new ResourceBundleProvider("messages");
CachingMessageProvider cached = new CachingMessageProvider(delegate, 1000, Duration.ofHours(1));
System.out.println("命中率: " + cached.getHitRate());
```

### 3.12 ReloadableMessageProvider

> 可热重载的消息提供者，监听文件系统变化自动刷新消息。实现 `AutoCloseable`。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `ReloadableMessageProvider(Path path)` | 从指定路径加载 |
| `ReloadableMessageProvider(Path path, Duration checkInterval)` | 指定检查间隔 |
| `Optional<String> getMessageTemplate(String key, Locale locale)` | 获取消息模板 |
| `Set<String> getKeys(Locale locale)` | 获取所有 key |
| `Set<Locale> getSupportedLocales()` | 获取支持的 Locale |
| `void refresh()` | 手动刷新 |
| `void startAutoReload()` | 启动自动重载 |
| `void stopAutoReload()` | 停止自动重载 |
| `void addReloadListener(Consumer<Set<Locale>> listener)` | 添加重载监听器 |
| `void close()` | 关闭并停止监听 |
| `boolean isWatching()` | 是否正在监听 |

**示例：**

```java
ReloadableMessageProvider provider = new ReloadableMessageProvider(
    Path.of("/config/i18n"), Duration.ofSeconds(30));
provider.startAutoReload();
provider.addReloadListener(locales -> System.out.println("重新加载: " + locales));

// 使用完毕关闭
provider.close();
```

### 3.15 AcceptHeaderLocaleResolver

> 基于 HTTP Accept-Language 请求头的 Locale 解析器，支持 quality 值和 Locale 匹配。

|------|------|
| `AcceptHeaderLocaleResolver(Supplier<String> headerSupplier)` | 指定 header 获取方式 |
| `AcceptHeaderLocaleResolver(Supplier<String> headerSupplier, Locale defaultLocale)` | 指定默认 Locale |
| `AcceptHeaderLocaleResolver(Supplier<String> headerSupplier, Locale defaultLocale, Set<Locale> supported)` | 指定支持的 Locale 集合 |
| `Locale resolve()` | 解析 Accept-Language 头 |
| `void setLocale(Locale locale)` | 设置覆盖 Locale |

| `Locale getDefaultLocale()` | 获取默认 Locale |
| `Set<Locale> getSupportedLocales()` | 获取支持的 Locale |

**内部类 `LocaleQuality`（record）：**

| 字段/方法 | 描述 |
|------|------|
| `Locale locale()` | 地区 |
| `double quality()` | 权重值（0.0~1.0） |

**示例：**

```java

```

### 3.18 NamedParameterFormatter

> 命名参数格式化器，支持 `${name}` 形式的占位符，可配置缺失参数行为。

**主要方法：**

| 方法 | 描述 |
|------|------|

| `String format(String template, Locale locale, Object... args)` | 位置参数格式化 |
| `String format(String template, Locale locale, Map<String, Object> params)` | 命名参数格式化 |
| `void clearCache()` | 清除缓存 |
| `String formatNested(String template, Map<String, Object> params, int maxDepth)` | 嵌套参数解析 |
| `boolean containsParameters(String template)` | 模板是否含参数占位符 |
| `Set<String> extractParameterNames(String template)` | 提取模板中所有参数名 |

**示例：**

```java
NamedParameterFormatter formatter = new NamedParameterFormatter();

```

### 3.24 ReloadableResourceBundle
