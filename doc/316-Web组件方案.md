# Web 组件方案

## 1. 组件概述

`opencode-base-web` 是一个轻量级 Web 工具组件，提供统一响应格式、分页封装、响应码管理、请求上下文、异常转换、响应加密等 RESTful API 开发常用能力。组件不依赖任何 Web 框架，纯 POJO / Record 设计，可与 Spring MVC、JAX-RS、Vert.x 等任意框架集成。

**核心特性：**
- 统一响应格式：`Result<T>` Record，标准化 API 响应结构
- 响应构建工具：`Results` 静态工厂 + `Builder` 模式
- 门面类：`OpenWeb` 提供结果、分页、上下文、编码、验证的一站式入口
- 分页封装：`PageRequest`、`PageResult<T>`、`PageInfo`、`Sort` 全部使用 Record
- 响应码体系：`ResultCode` 接口 + `CommonResultCode` 预定义 60+ 响应码
- 请求上下文：`RequestContext` / `UserContext` Record + `RequestContextHolder` ThreadLocal 管理
- 异常体系：`OpenWebException` / `OpenBizException` + `ExceptionConverter` 自动转换
- 响应加密：`ResultEncryptor` SPI + `AesResultEncryptor` AES 加密实现
- SPI 扩展：`ResultCodeProvider`、`ResultCustomizer` 支持自定义扩展
- Web 工具：URL 编解码、IP 验证、Email 验证、Base64 编解码

## 3. 核心 API

### 3.1 Result

> 统一响应结果 Record，标准化 API 响应格式，包含状态码、消息、数据、是否成功、时间戳和追踪 ID。不可变，线程安全。

**Record 字段：**

| 字段 | 类型 | 描述 |
|------|------|------|
| `code` | `String` | 响应码 |
| `message` | `String` | 响应消息 |
| `data` | `T` | 响应数据 |
| `success` | `boolean` | 是否成功 |
| `timestamp` | `Instant` | 时间戳 |
| `traceId` | `String` | 追踪 ID（可选） |

**主要方法：**

| 方法 | 描述 |
|------|------|
| `static <T> Result<T> ok()` | 成功响应（无数据） |
| `static <T> Result<T> ok(T data)` | 成功响应（带数据） |
| `static <T> Result<T> ok(String message, T data)` | 成功响应（带消息和数据） |
| `static <T> Result<T> fail(String code, String message)` | 失败响应 |
| `static <T> Result<T> fail(ResultCode resultCode)` | 失败响应（使用响应码） |
| `static <T> Result<T> fail(ResultCode resultCode, String message)` | 失败响应（响应码+自定义消息） |
| `static <T> Result<T> fail(Throwable throwable)` | 从异常创建失败响应 |
| `boolean isFailed()` | 判断是否失败 |
| `T getDataOrThrow()` | 获取数据（失败时抛 OpenBizException） |
| `T getDataOrDefault(T defaultValue)` | 获取数据（失败时返回默认值） |
| `<R> Result<R> map(Function<T, R> mapper)` | 转换数据类型 |
| `<R> Result<R> flatMap(Function<T, Result<R>> mapper)` | 链式处理 |
| `Result<T> onSuccess(Consumer<T> action)` | 成功时执行回调 |
| `Result<T> onFailure(BiConsumer<String, String> action)` | 失败时执行回调 |
| `Result<T> withTraceId(String newTraceId)` | 替换 traceId |
| `Result<T> withMessage(String newMessage)` | 替换消息 |

**示例：**

```java
// 成功响应
Result<User> result = Result.ok(user);

// 失败响应
Result<Void> fail = Result.fail(CommonResultCode.NOT_FOUND);

// 链式处理
Result<String> name = result
    .map(User::getName)
    .onSuccess(n -> log.info("用户: {}", n))
    .onFailure((code, msg) -> log.warn("失败: {} {}", code, msg));

// 获取数据
User u = result.getDataOrThrow();
User u2 = result.getDataOrDefault(defaultUser);
```

**Builder 方法：**

| 方法 | 描述 |
|------|------|
| `Builder<T> code(String code)` | 设置响应码 |
| `Builder<T> code(ResultCode resultCode)` | 使用响应码枚举 |
| `Builder<T> message(String message)` | 设置消息 |
| `Builder<T> data(T data)` | 设置数据 |

**示例：**

```java

    .code(CommonResultCode.SUCCESS)
    .message("查询成功")
    .data(user)
    .traceId("trace-123")
    .build();

// 分页响应
Result<PageResult<User>> page = Results.page(users, 100L, 1, 10);
```

### 3.3 ResultCode

> 响应码接口，所有响应码枚举应实现此接口，支持自定义扩展。

**主要方法：**

| 方法 | 描述 |
|------|------|
| `String getCode()` | 获取响应码 |
| `String getMessage()` | 获取响应消息 |
| `int getHttpStatus()` | 获取 HTTP 状态码 |
| `default boolean isSuccess()` | 判断是否成功码 |
| `static ResultCode of(String code, String message, int httpStatus)` | 创建自定义响应码 |

**示例：**

```java
// 自定义业务响应码
public enum BizResultCode implements ResultCode {
    ORDER_TIMEOUT("BIZ_001", "订单超时", 400),
    STOCK_INSUFFICIENT("BIZ_002", "库存不足", 400);

    private final String code;
    private final String message;
    private final int httpStatus;

    @Override
    public String getCode() { return code; }
    @Override
    public String getMessage() { return message; }
    @Override
    public int getHttpStatus() { return httpStatus; }
}

// 动态创建
ResultCode custom = ResultCode.of("CUSTOM_001", "自定义错误", 400);
```

### 3.4 CommonResultCode

> 通用响应码枚举，预定义 60+ 常用响应码，涵盖成功、客户端错误、参数错误、认证错误、文件错误、限流错误、服务端错误、业务错误、数据库错误、并发错误、第三方错误等。

**编码规范：**

| 分类 | 编码前缀 | 说明 |
|------|----------|------|
| 成功 | `00xxx` | 成功类响应 |
| 客户端错误 | `A0xxx` | HTTP 4xx 对应 |
| 参数错误 | `A1xxx` | 参数校验失败 |
| 认证错误 | `A2xxx` | 认证/授权失败 |
| 文件错误 | `A3xxx` | 文件操作失败 |
| 限流错误 | `A4xxx` | 速率限制 |
| 服务端错误 | `B0xxx` | HTTP 5xx 对应 |
| 业务错误 | `B1xxx` | 业务逻辑失败 |
| 数据库错误 | `B2xxx` | 数据库相关 |
| 并发错误 | `B3xxx` | 并发冲突 |
| 第三方错误 | `C0xxx` | 第三方服务 |
| 支付错误 | `C1xxx` | 支付相关 |
| 消息错误 | `C2xxx` | 消息发送失败 |

**主要方法：**

| 方法 | 描述 |
|------|------|
| `String getCode()` | 获取响应码 |
| `String getMessage()` | 获取响应消息 |
| `int getHttpStatus()` | 获取 HTTP 状态码 |
| `static CommonResultCode fromCode(String code)` | 按编码查找 |
| `static CommonResultCode fromCode(String code, CommonResultCode defaultCode)` | 按编码查找（带默认值） |

**常用响应码示例：**

| 枚举值 | 编码 | 消息 | HTTP 状态 |
|--------|------|------|-----------|
| `SUCCESS` | `00000` | 成功 | 200 |

| `FORBIDDEN` | `A0403` | 禁止访问 | 403 |

| `TOO_MANY_REQUESTS` | `A0429` | 请求过多 | 429 |
| `INTERNAL_ERROR` | `B0500` | 服务器内部错误 | 500 |
| `BUSINESS_ERROR` | `B1001` | 业务错误 | 500 |
| `DATA_NOT_FOUND` | `B1002` | 数据不存在 | 404 |
| `DATA_DUPLICATE` | `B1003` | 数据重复 | 409 |
| `THIRD_PARTY_ERROR` | `C0001` | 第三方错误 | 502 |

### 3.11 RequestContextHolder

> 请求上下文持有者，基于 ThreadLocal 管理请求上下文，支持异步传递（wrap 方法）。

**主要方法：**

| 方法 | 描述 |
|------|------|

| `static RequestContext getContext()` | 获取上下文 |
| `static RequestContext getOrCreate(Supplier<RequestContext>)` | 获取或创建上下文 |
| `static void clear()` | 清除上下文 |
| `static boolean hasContext()` | 是否有上下文 |
| `static String getTraceId()` | 获取追踪 ID |
| `static UserContext getUser()` | 获取用户上下文 |
| `static String getUserId()` | 获取用户 ID |
| `static Runnable wrap(Runnable runnable)` | 包装 Runnable（传递上下文到新线程） |
| `static <T> Callable<T> wrap(Callable<T> callable)` | 包装 Callable |
| `static <T> Supplier<T> wrap(Supplier<T> supplier)` | 包装 Supplier |
| `static void execute(RequestContext ctx, Runnable runnable)` | 在指定上下文中执行 |
| `static <T> T execute(RequestContext ctx, Supplier<T> supplier)` | 在指定上下文中执行并返回结果 |

**示例：**

```java
// 设置上下文
RequestContextHolder.setContext(RequestContext.of("trace-001"));
try {
    String traceId = RequestContextHolder.getTraceId();
    // ...
} finally {
    RequestContextHolder.clear();
}

// 异步传递上下文
Runnable wrapped = RequestContextHolder.wrap(() -> {
    // 子线程中可获取父线程上下文
    String traceId = RequestContextHolder.getTraceId();
});
executor.submit(wrapped);
```
