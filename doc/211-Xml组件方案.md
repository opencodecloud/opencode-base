# Xml 组件方案

## 1. 组件概述

### 1.1 设计目标

`opencode-base-xml` 是一个基于 JDK 内置 XML API 的轻量级 XML 处理组件，提供 DOM、SAX、StAX 等多种解析方式的统一封装，以及 XPath 查询、XML 与 Bean 互转、XSLT 转换、Schema/DTD 验证等功能。

### 1.2 核心特性

- **统一门面 API**：`OpenXml` 提供解析、构建、查询、验证、绑定、转换等一站式入口
- **多解析模式**：支持 DOM、SAX、StAX，按场景选择最优解析方式
- **XPath 查询**：强大的 XPath 表达式支持，包含命名空间感知查询
- **Bean 绑定**：基于注解的 XML 与 Java Bean 自动互转（JAXB 风格）
- **流式构建**：`XmlBuilder` 和 `ElementBuilder` 提供链式 API 构建 XML 文档
- **命名空间支持**：完整的命名空间处理，包括 `OpenNamespaceContext` 和 `NamespaceUtil`
- **安全解析**：默认防止 XXE 攻击，`XmlSecurity` 和 `SecureParserFactory` 提供安全配置
- **验证功能**：支持 Schema (XSD) 验证、DTD 验证、格式良好性检查
- **XSLT 转换**：内置 `XsltTransformer` 支持 XSLT 样式表转换
- **JDK 25 原生**：无第三方依赖，仅使用 JDK 内置 API

### 1.4 解析模式选择指南

| 解析模式 | 内存占用 | 性能 | 随机访问 | 适用场景 |
|---------|---------|------|---------|---------|
| **DOM** | 高（全量加载） | 中等 | 支持 | 小文件（<10MB）、需要修改、XPath 查询 |
| **SAX** | 低（事件驱动） | 快 | 不支持 | 大文件、只读、顺序处理 |
| **StAX** | 低（拉模式） | 最快 | 不支持 | 大文件、需要控制解析流程、流式写入 |

## 3. 核心 API

### 3.1 OpenXml -- 统一门面

`OpenXml` 是整个 XML 模块的统一入口，提供解析、构建、XPath 查询、验证、格式化、绑定、转换等全部核心功能的静态方法。

```java
public final class OpenXml {

    // === 解析 ===
    public static XmlDocument parse(String xml);
    public static XmlDocument parseFile(Path path);
    public static XmlDocument parse(InputStream input);

    // === 构建 ===
    public static XmlBuilder builder(String rootName);
    public static XmlBuilder builder(String namespaceUri, String rootName);
    public static ElementBuilder element(String name);

    // === XPath 查询 ===
    public static String xpath(XmlDocument doc, String xpath);
    public static String xpath(XmlDocument doc, String xpath, Map<String, String> namespaces);
    public static List<XmlElement> xpathElements(XmlDocument doc, String xpath);
    public static String xpath(XmlElement element, String xpath);

    // === 验证 ===
    public static boolean isWellFormed(String xml);
    public static ValidationResult validateWellFormedness(String xml);
    public static ValidationResult validateSchema(String xml, Path schemaPath);
    public static SchemaValidator schemaValidator(Path schemaPath);

    // === 格式化 ===
    public static String format(String xml, int indent);
    public static String format(String xml);           // 默认 2 空格缩进
    public static String minify(String xml);

    // === XSLT 转换 ===
    public static String xslt(String xml, Path xsltPath);
    public static XsltTransformer xsltTransformer(Path xsltPath);
    public static XsltTransformer xsltTransformer(String xslt);

    // === Bean 绑定 (marshal/unmarshal) ===
    public static <T> T unmarshal(String xml, Class<T> clazz);
    public static <T> T unmarshal(XmlDocument document, Class<T> clazz);
    public static String marshal(Object obj);
    public static String marshal(Object obj, int indent);
    public static XmlBinder binder();

    // === SAX/StAX 流式处理 ===
    public static SaxParser saxParser();
    public static StaxReader staxReader(String xml);
    public static StaxReader staxReader(Path path);
    public static StaxWriter staxWriter();
    public static StaxWriter staxWriter(Path path);

    // === 命名空间 ===
    public static OpenNamespaceContext namespaceContext();
    public static OpenNamespaceContext namespaceContext(XmlDocument document);
    public static Map<String, String> extractNamespaces(XmlDocument document);
}
```

**使用示例：**

```java
// 解析 XML 字符串
XmlDocument doc = OpenXml.parse("<users><user id='1'>Alice</user></users>");

// 从文件解析
XmlDocument doc = OpenXml.parseFile(Path.of("config.xml"));

// XPath 查询
String name = OpenXml.xpath(doc, "//user/text()");
List<XmlElement> users = OpenXml.xpathElements(doc, "//user");

// 验证 Schema
ValidationResult result = OpenXml.validateSchema(xml, Path.of("schema.xsd"));

// 格式化 XML
String formatted = OpenXml.format(xml, 4);

// XSLT 转换
String result = OpenXml.xslt(xml, Path.of("transform.xslt"));

// Bean 绑定
User user = OpenXml.unmarshal(xml, User.class);
String xml = OpenXml.marshal(user);

// 构建 XML
XmlDocument doc = OpenXml.builder("root")
    .element("name", "Alice")
    .element("age", 30)
    .build();
```

## 4. 线程安全与性能

### 4.1 线程安全

| 组件 | 线程安全 | 说明 |
|------|---------|------|
| `OpenXml` | 是 | 无状态静态方法 |
| `OpenXPath` | 是 | 无状态静态方法 |
| `XmlDocument` | 否 | DOM 节点非线程安全，每次解析创建新实例 |
| `XmlElement` | 否 | DOM 节点非线程安全 |
| `XPathQuery` | 否 | 构建器模式，每次新建 |
| `XmlBuilder` | 否 | 构建器模式，每次新建 |
| `XmlBinder` | 是 | 实例可安全复用 |
| `XsltTransformer` | 条件安全 | `Templates` 可复用，每次 `transform()` 内部创建新 `Transformer` |
| `SchemaValidator` | 是 | `Schema` 对象线程安全 |
| `SecureParserFactory` | 是 | Factory 对象可缓存复用 |

### 4.2 性能优化建议

```java
// 1. 复用 Schema 验证器（Schema 编译一次）
SchemaValidator validator = SchemaValidator.of(schemaPath);
// 后续多次调用 validator.validate(...)

// 2. 复用 XSLT 转换器（Templates 编译一次）
XsltTransformer transformer = XsltTransformer.of(xsltPath);
// 后续多次调用 transformer.transform(...)

// 3. 大文件使用 StAX 流式处理
try (StaxReader reader = StaxReader.of(Path.of("large.xml"))) {
    reader.onElement("item", (name, attrs) -> process(attrs))
          .read();
}

// 4. 复用 XmlBinder 实例
XmlBinder binder = XmlBinder.create()
    .registerAdapter(new DateAdapter.LocalDateAdapter());
// 后续多次调用 binder.unmarshal(...)
```
