# Geo 组件方案

## 1. 组件概述

### 1.1 设计目标

`opencode-base-geo` 模块提供地理信息处理能力，涵盖坐标系转换、距离计算、地理围栏、GeoHash 编码、行政区划、坐标验证与隐私保护。

**核心特性：**
- 坐标系转换（WGS84 / GCJ02 / BD09 互转）
- 距离计算（Haversine 快速算法 / Vincenty 精确算法）
- 地理围栏（圆形 / 矩形 / 多边形 / 跨日期线）
- GeoHash 编解码与邻域查询
- 行政区划管理（Region / RegionService）
- 方位角与目的地计算
- 坐标验证（范围检查 / 中国境内检查 / GeoHash 格式检查）
- 坐标隐私保护（随机偏移 / 精度降级 / GeoHash 网格对齐）
- 安全围栏服务（时间戳验证 / 速度检测 / 防位置欺骗）

## 3. 核心 API

### 3.1 Coordinate -- 坐标实体

不可变 record，表示一个地理坐标点，包含经度、纬度和坐标系信息。

```java
public record Coordinate(
    double longitude,          // 经度
    double latitude,           // 纬度
    CoordinateSystem system    // 坐标系
) { ... }
```

**主要方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `wgs84(double lng, double lat)` | 经度, 纬度 | `Coordinate` | 创建 WGS84 坐标 |
| `gcj02(double lng, double lat)` | 经度, 纬度 | `Coordinate` | 创建 GCJ02 坐标（火星坐标系） |
| `bd09(double lng, double lat)` | 经度, 纬度 | `Coordinate` | 创建 BD09 坐标（百度坐标系） |
| `to(CoordinateSystem target)` | 目标坐标系 | `Coordinate` | 转换到目标坐标系 |
| `distanceTo(Coordinate other)` | 另一个坐标 | `double` | 计算到另一点的距离（米） |
| `toGeoHash(int precision)` | 精度 1-12 | `String` | 转换为 GeoHash 字符串 |
| `isValid()` | 无 | `boolean` | 检查坐标是否有效（范围合法、非 NaN、非 Infinite） |
| `isInChina()` | 无 | `boolean` | 检查坐标是否在中国境内 |

**代码示例：**

```java
// 创建坐标
Coordinate beijing = Coordinate.wgs84(116.4074, 39.9042);
Coordinate shanghai = Coordinate.wgs84(121.4737, 31.2304);

// 坐标系转换
Coordinate gcj02 = beijing.to(CoordinateSystem.GCJ02);
Coordinate bd09 = beijing.to(CoordinateSystem.BD09);

// 距离计算
double distance = beijing.distanceTo(shanghai); // 约 1068 公里

// GeoHash
String hash = beijing.toGeoHash(8); // "wx4g0bce"

// 验证
boolean valid = beijing.isValid();     // true
boolean inChina = beijing.isInChina(); // true
```

### 3.2 CoordinateSystem -- 坐标系枚举

```java
public enum CoordinateSystem {
    WGS84,  // GPS 国际标准坐标系
    GCJ02,  // 中国国测局坐标系（火星坐标系）
    BD09    // 百度坐标系
}
```

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `getCode()` | `String` | 获取坐标系代码 |
| `getDescription()` | `String` | 获取坐标系描述 |

### 3.3 OpenGeo -- 门面入口类

全静态方法的门面类，提供距离计算、坐标转换、围栏判断、GeoHash、方位角等一站式 API。

**常量：**
- `EARTH_RADIUS = 6371000.0` -- 地球平均半径（米）

**距离计算方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `distance(Coordinate c1, Coordinate c2)` | 两个坐标 | `double` | Haversine 算法计算距离（米） |
| `distance(double lng1, double lat1, double lng2, double lat2)` | 经纬度 | `double` | 根据经纬度计算距离（米） |
| `distance(Coordinate c1, Coordinate c2, DistanceAccuracy accuracy)` | 坐标, 精度级别 | `double` | 指定算法精度计算距离 |
| `distancePrecise(Coordinate c1, Coordinate c2)` | 两个坐标 | `double` | Vincenty 精确算法（误差 ~0.5mm） |
| `formatDistance(double meters)` | 距离（米） | `String` | 格式化距离（"500米" / "1.5公里"） |

**坐标转换方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `wgs84ToGcj02(double lng, double lat)` | 经度, 纬度 | `Coordinate` | WGS84 转 GCJ02 |
| `gcj02ToBd09(double lng, double lat)` | 经度, 纬度 | `Coordinate` | GCJ02 转 BD09 |
| `bd09ToGcj02(double lng, double lat)` | 经度, 纬度 | `Coordinate` | BD09 转 GCJ02 |
| `wgs84ToBd09(double lng, double lat)` | 经度, 纬度 | `Coordinate` | WGS84 转 BD09 |
| `bd09ToWgs84(double lng, double lat)` | 经度, 纬度 | `Coordinate` | BD09 转 WGS84 |
| `gcj02ToWgs84(double lng, double lat)` | 经度, 纬度 | `Coordinate` | GCJ02 转 WGS84 |

**围栏判断方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `inCircle(Coordinate point, Coordinate center, double radius)` | 点, 圆心, 半径（米） | `boolean` | 判断是否在圆形围栏内 |
| `inPolygon(Coordinate point, List<Coordinate> polygon)` | 点, 多边形顶点 | `boolean` | 判断是否在多边形内 |
| `inRectangle(Coordinate point, Coordinate sw, Coordinate ne)` | 点, 西南角, 东北角 | `boolean` | 判断是否在矩形内 |

**GeoHash 方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `geoHash(double lng, double lat, int precision)` | 经度, 纬度, 精度 | `String` | 编码 GeoHash |
| `fromGeoHash(String hash)` | GeoHash 字符串 | `Coordinate` | 解码为坐标 |
| `geoHashNeighbors(String hash)` | GeoHash 字符串 | `List<String>` | 获取 8 个相邻 GeoHash |
| `geoHashBoundingBox(String hash)` | GeoHash 字符串 | `double[]` | 获取 GeoHash 边界框 |

**导航与辅助方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `bearing(Coordinate from, Coordinate to)` | 起点, 终点 | `double` | 计算方位角（度，0-360） |
| `destination(Coordinate start, double distance, double bearing)` | 起点, 距离（米）, 方位角 | `Coordinate` | 根据起点、距离和方位角计算终点 |
| `midpoint(Coordinate c1, Coordinate c2)` | 两个坐标 | `Coordinate` | 计算两点中点 |
| `isInChina(double lng, double lat)` | 经度, 纬度 | `boolean` | 判断是否在中国境内 |
| `isValidCoordinate(double lng, double lat)` | 经度, 纬度 | `boolean` | 检查坐标是否有效 |

**代码示例：**

```java
// 距离计算
Coordinate beijing = Coordinate.wgs84(116.4074, 39.9042);
Coordinate shanghai = Coordinate.wgs84(121.4737, 31.2304);
double dist = OpenGeo.distance(beijing, shanghai);
System.out.println(OpenGeo.formatDistance(dist));  // "1068.0公里"

// 精确距离
double precise = OpenGeo.distancePrecise(beijing, shanghai);

// 坐标转换
Coordinate gcj02 = OpenGeo.wgs84ToGcj02(116.4074, 39.9042);
Coordinate bd09 = OpenGeo.gcj02ToBd09(gcj02.longitude(), gcj02.latitude());

// GeoHash
String hash = OpenGeo.geoHash(116.4074, 39.9042, 8);
Coordinate decoded = OpenGeo.fromGeoHash(hash);
List<String> neighbors = OpenGeo.geoHashNeighbors(hash);

// 围栏判断
boolean inCircle = OpenGeo.inCircle(shanghai, beijing, 2000000); // 2000 公里
boolean inChina = OpenGeo.isInChina(116.4074, 39.9042);

// 方位角与目的地
double bearing = OpenGeo.bearing(beijing, shanghai);
Coordinate dest = OpenGeo.destination(beijing, 100000, 135); // 东南方向 100 公里
Coordinate mid = OpenGeo.midpoint(beijing, shanghai);
```

- `METERS_PER_DEGREE = 111319.9` -- 每度约对应的米数

**边界框方法：**

### 3.5 CoordinateUtil -- 坐标转换工具类

提供坐标系之间的转换功能，以 GCJ02 为中间桥梁实现任意两个坐标系之间的转换。

**主要方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `transform(Coordinate coord, CoordinateSystem target)` | 原坐标, 目标坐标系 | `Coordinate` | 坐标系转换（自动路由） |
| `wgs84ToGcj02(Coordinate wgs84)` | WGS84 坐标 | `Coordinate` | WGS84 转 GCJ02 |
| `gcj02ToBd09(Coordinate gcj02)` | GCJ02 坐标 | `Coordinate` | GCJ02 转 BD09 |
| `bd09ToGcj02(Coordinate bd09)` | BD09 坐标 | `Coordinate` | BD09 转 GCJ02 |
| `gcj02ToWgs84(Coordinate gcj02)` | GCJ02 坐标 | `Coordinate` | GCJ02 转 WGS84 |

**转换逻辑说明：**
- 相同坐标系直接返回原对象
- 不同坐标系先转换到 GCJ02，再从 GCJ02 转换到目标坐标系
- 中国境外坐标不做偏移处理

**代码示例：**

```java
// 通过 Coordinate.to() 调用（推荐）
Coordinate wgs84 = Coordinate.wgs84(116.4074, 39.9042);
Coordinate gcj02 = wgs84.to(CoordinateSystem.GCJ02);
Coordinate bd09 = wgs84.to(CoordinateSystem.BD09);

// 直接使用工具类
Coordinate result = CoordinateUtil.transform(wgs84, CoordinateSystem.BD09);

// 往返转换
Coordinate roundTrip = bd09.to(CoordinateSystem.WGS84);
// roundTrip 与 wgs84 误差极小（< 0.0001 度）
```

## 4. 距离计算

### 4.1 DistanceCalculator -- 距离计算器接口

```java
public interface DistanceCalculator {
    double calculate(Coordinate c1, Coordinate c2);
}
```

### 4.2 HaversineCalculator -- Haversine 算法

快速距离计算，适合大多数应用场景，误差约 0.5%。

```java
HaversineCalculator.INSTANCE.calculate(c1, c2);
```

### 4.3 VincentyCalculator -- Vincenty 算法

高精度距离计算，适合测绘等高精度场景，误差约 0.5mm。

```java
VincentyCalculator.INSTANCE.calculate(c1, c2);
```

### 4.4 DistanceCalculatorFactory -- 计算器工厂

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `create(DistanceAccuracy accuracy)` | 精度级别 | `DistanceCalculator` | 根据精度创建计算器 |
| `getDefault()` | 无 | `DistanceCalculator` | 获取默认（Haversine）计算器 |
| `getPrecise()` | 无 | `DistanceCalculator` | 获取精确（Vincenty）计算器 |

**DistanceAccuracy 枚举：**

| 值 | 算法 | 精度 | 适用场景 |
|----|------|------|----------|
| `FAST` | Haversine | ~0.5% 误差 | 导航、打车、外卖、大多数场景 |
| `PRECISE` | Vincenty | ~0.5mm 误差 | 测绘、精确边界计算 |

**代码示例：**

```java
// 默认快速算法
double dist = OpenGeo.distance(c1, c2);

// 指定精度
double fast = OpenGeo.distance(c1, c2, DistanceAccuracy.FAST);
double precise = OpenGeo.distance(c1, c2, DistanceAccuracy.PRECISE);

// 直接使用工厂
DistanceCalculator calculator = DistanceCalculatorFactory.create(DistanceAccuracy.PRECISE);
```

## 5. 地理围栏

### 5.1 GeoFence -- 围栏接口

```java
public interface GeoFence {
    boolean contains(Coordinate point);
}
```

### 5.2 CircleFence -- 圆形围栏

```java
public record CircleFence(Coordinate center, double radius) implements GeoFence {
    public boolean contains(Coordinate point);  // 距离 <= 半径即在围栏内
}
```

**参数验证：** 构造时检查 center 非 null、radius > 0。

### 5.3 RectangleFence -- 矩形围栏

```java
public record RectangleFence(Coordinate southwest, Coordinate northeast) implements GeoFence {
    public boolean contains(Coordinate point);    // 自动处理跨日期线
    public boolean crossesDateLine();             // 是否跨越国际日期线
    public double widthDegrees();                 // 宽度（度）
    public double heightDegrees();                // 高度（度）
    public Coordinate center();                   // 中心点
}
```

**跨日期线检测：** 当 `southwest.longitude() > northeast.longitude()` 时自动识别为跨日期线。

### 5.4 PolygonFence -- 多边形围栏

```java
public record PolygonFence(List<Coordinate> vertices) implements GeoFence {
    public boolean contains(Coordinate point);          // Ray-casting 算法 + 精确边界处理
    public int vertexCount();                           // 顶点数量
    public RectangleFence getBoundingBox();             // 最小外接矩形
}
```

**特点：**
- 使用 Ray-casting 算法判断点是否在多边形内
- 对边界点（顶点和边上的点）做精确处理，保证返回 `true`
- 构造时验证至少 3 个顶点

### 5.5 CrossDateLineFence -- 跨日期线围栏

```java
public record CrossDateLineFence(Coordinate southwest, Coordinate northeast) implements GeoFence {
    public boolean contains(Coordinate point);
    public boolean crossesDateLine();
}
```

**代码示例：**

```java
// 圆形围栏
Coordinate center = Coordinate.wgs84(116.4074, 39.9042);
GeoFence circle = new CircleFence(center, 1000); // 1km 半径
boolean inside = circle.contains(somePoint);

// 矩形围栏
GeoFence rect = new RectangleFence(
    Coordinate.wgs84(116.0, 39.0),
    Coordinate.wgs84(117.0, 40.0)
);

// 多边形围栏
List<Coordinate> vertices = List.of(
    Coordinate.wgs84(116.0, 39.0),
    Coordinate.wgs84(117.0, 39.0),
    Coordinate.wgs84(117.0, 40.0),
    Coordinate.wgs84(116.0, 40.0)
);
GeoFence polygon = new PolygonFence(vertices);
boolean inPolygon = polygon.contains(Coordinate.wgs84(116.5, 39.5));

// 跨日期线矩形
GeoFence crossDateLine = new RectangleFence(
    Coordinate.wgs84(170.0, -10.0),
    Coordinate.wgs84(-170.0, 10.0)
);
```

## 6. GeoHash

### 6.1 GeoHash 接口

```java
public interface GeoHash {
```

### 6.2 GeoHashEncoder -- 编解码实现

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|

| `neighbors(String geoHash)` | GeoHash 字符串 | `List<String>` | 获取 8 个相邻 GeoHash |
| `getBoundingBox(String geoHash)` | GeoHash 字符串 | `double[]` | 获取边界框 [minLat, minLng, maxLat, maxLng] |

`INSTANCE` 单例可直接使用。

### 6.3 GeoHashUtil -- 工具类

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `encode(double lat, double lng, int precision)` | 纬度, 经度, 精度 | `String` | 编码 GeoHash |
| `encode(double lat, double lng)` | 纬度, 经度 | `String` | 使用默认精度编码 |
| `decode(String geoHash)` | GeoHash | `double[]` | 解码 GeoHash |
| `neighbors(String geoHash)` | GeoHash | `List<String>` | 获取 8 个相邻 GeoHash |
| `getBoundingBox(String geoHash)` | GeoHash | `double[]` | 获取边界框 |
| `getPrecisionForRadius(double radiusKm)` | 搜索半径（公里） | `int` | 根据半径推荐合适精度 |
| `isValid(String geoHash)` | GeoHash 字符串 | `boolean` | 检查 GeoHash 格式是否有效 |

**GeoHash 精度参考：**

| 精度 | 网格大小 | 适用场景 |
|------|----------|----------|
| 1 | ~5000km | 洲级别 |
| 2 | ~1250km | 大区域 |
| 3 | ~156km | 省/大城市 |
| 4 | ~39km | 城市 |
| 5 | ~5km | 区/县 |
| 6 | ~1.2km | 街道/社区 |
| 7 | ~153m | 建筑物 |
| 8 | ~38m | 精确位置 |
| 9 | ~5m | 高精度 |

**代码示例：**

```java

// 解码

// 邻域

// 根据搜索半径选择精度
int precision = GeoHashUtil.getPrecisionForRadius(5.0); // 5km -> 精度 5

// 通过 OpenGeo 使用
```

### 13.2 附近搜索示例

```java
public List<POI> findNearbyPOIs(Coordinate center, double radiusKm, List<POI> allPOIs) {
    // 1. 计算合适的 GeoHash 精度
    int precision = GeoHashUtil.getPrecisionForRadius(radiusKm);
    String centerHash = center.toGeoHash(precision);

    // 2. 获取周围 9 个网格
    Set<String> hashes = new HashSet<>(OpenGeo.geoHashNeighbors(centerHash));
    hashes.add(centerHash);

    // 3. 粗筛：GeoHash 前缀匹配
    List<POI> candidates = allPOIs.stream()
        .filter(poi -> hashes.stream()
            .anyMatch(h -> poi.getGeoHash().startsWith(h)))
        .toList();

    // 4. 精确筛选
    double radiusMeters = radiusKm * 1000;
    return candidates.stream()
        .filter(poi -> center.distanceTo(poi.getCoordinate()) <= radiusMeters)
        .toList();
}
```
