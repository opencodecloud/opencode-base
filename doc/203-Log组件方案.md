# Log 组件方案

## 1. 组件概述

### 1.1 设计目标

`opencode-base-log` 是一个轻量级日志门面组件，通过 SPI 机制适配主流日志框架（SLF4J、Log4j2、JUL），提供统一的日志 API 和增强功能。组件不重新实现日志引擎，而是提供简洁的门面 API 和增强功能。

### 1.2 核心特性

- **统一门面 API**：`OpenLog.info()`、`OpenLog.error()` 等静态方法，零配置即用
- **SPI 可插拔引擎**：支持 SLF4J、Log4j2、JUL 自由切换
- **Lambda 延迟求值**：`OpenLog.debug(() -> expensiveOperation())`，避免无效计算
- **结构化日志**：支持 MDC/NDC，便于日志追踪和分析
- **性能监控日志**：内置方法耗时统计、慢操作告警
- **审计日志**：独立的审计日志记录，支持登录/登出/数据变更等场景
- **JDK 25 增强**：Virtual Thread 上下文传播、ScopedValues 支持
- **结构化 JSON 日志**：原生 JSON 格式输出，适配 ELK/Loki
- **日志脱敏**：敏感信息自动脱敏（手机号、邮箱、身份证、密码等）
- **采样日志**：高频日志采样控制（按比例、按时间、按计数）
- **条件日志**：环境感知日志（仅开发/测试/生产）、限流日志、首次日志
- **异常增强**：异常根因分析、去重、限流、签名计算
- **日志指标**：日志与指标集成，自动计数与计时

## 3. 核心 API

### 3.1 LogLevel - 日志级别枚举

定义标准日志级别，按严重程度从低到高排列。

```java
public enum LogLevel {
    TRACE(1, "TRACE"),
    DEBUG(2, "DEBUG"),
    INFO(3, "INFO"),
    WARN(4, "WARN"),
    ERROR(5, "ERROR"),
    OFF(6, "OFF");
}
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getLevel()` | - | `int` | 获取级别数值 |
| `getName()` | - | `String` | 获取级别名称 |
| `isGreaterOrEqual(LogLevel other)` | `other` - 比较目标 | `boolean` | 判断是否大于等于指定级别 |
| `isEnabledFor(LogLevel threshold)` | `threshold` - 阈值级别 | `boolean` | 判断是否在阈值级别下启用 |
| `fromName(String name)` | `name` - 级别名称 | `LogLevel` | 从名称解析级别 |
| `fromNameOrDefault(String name, LogLevel defaultLevel)` | `name` - 名称；`defaultLevel` - 默认值 | `LogLevel` | 从名称解析，无效时返回默认值 |

### 3.2 OpenLog - 静态门面

统一日志门面，提供零配置即用的静态方法。自动推断调用类获取 Logger。

```java
// 获取Logger
Logger log = OpenLog.get();                    // 自动推断调用类
Logger log = OpenLog.get(MyClass.class);       // 指定类
Logger log = OpenLog.get("com.example");       // 指定名称

// 各级别日志输出
OpenLog.trace("详细追踪信息");
OpenLog.debug("调试信息: {}", debugValue);
OpenLog.debug(() -> "昂贵计算: " + computeExpensiveValue());
OpenLog.info("应用启动成功");
OpenLog.info("用户 {} 登录，IP: {}", userId, ip);
OpenLog.warn("配置项已废弃: {}", key);
OpenLog.warn("操作异常", exception);
OpenLog.error("处理失败", exception);
OpenLog.error(exception);                      // 仅异常
OpenLog.error(() -> "延迟消息", exception);     // Lambda + 异常

// 带Marker
OpenLog.info(marker, "带标记的日志");
OpenLog.error(marker, "错误信息", exception);

// 级别判断
if (OpenLog.isDebugEnabled()) {
    OpenLog.debug("仅在DEBUG启用时执行");
}
```

| `debug(String message)` | `message` - 消息 | `void` | 输出DEBUG日志 |
| `debug(String format, Object... args)` | `format` - 格式；`args` - 参数 | `void` | 输出DEBUG日志（带参数） |

### 3.4 Logger - 日志记录器接口

Logger 接口定义了完整的日志记录方法，每个级别（TRACE/DEBUG/INFO/WARN/ERROR）均支持以下重载：

- `void xxx(String message)` - 普通消息
- `void xxx(String format, Object... args)` - 参数化消息
- `void xxx(String message, Throwable throwable)` - 带异常
- `void xxx(Supplier<String> messageSupplier)` - Lambda延迟求值
- `void xxx(Marker marker, String message)` - 带标记
- `void xxx(Marker marker, String format, Object... args)` - 带标记+参数
- `boolean isXxxEnabled()` - 级别判断
- `boolean isXxxEnabled(Marker marker)` - 带标记的级别判断

通用方法：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getName()` | - | `String` | 获取Logger名称 |
| `isEnabled(LogLevel level)` | `level` - 级别 | `boolean` | 判断指定级别是否启用 |
| `log(LogLevel level, String message)` | `level` - 级别；`message` - 消息 | `void` | 输出指定级别的日志 |
| `log(LogLevel level, String format, Object... args)` | `level` - 级别；`format` - 格式；`args` - 参数 | `void` | 输出指定级别的参数化日志 |
| `log(LogLevel level, String message, Throwable throwable)` | `level` - 级别；`message` - 消息；`throwable` - 异常 | `void` | 输出指定级别的异常日志 |

## 4. 上下文管理

### 4.1 MDC - 映射诊断上下文

在日志中添加键值对上下文信息（基于 ThreadLocal）。

```java
// 基本操作
MDC.put("requestId", "req-12345");
MDC.put("userId", "user-001");
String value = MDC.get("requestId");
MDC.remove("requestId");
MDC.clear();

// 获取/设置上下文副本
Map<String, String> ctx = MDC.getCopyOfContextMap();
MDC.setContextMap(ctx);

// 在指定上下文中执行
MDC.runWith("taskId", "task-001", () -> {
    // 此作用域内 taskId 生效
    processTask();
});

String result = MDC.callWith("taskId", "task-001", () -> {
    return processTask();
});

// 使用作用域（AutoCloseable）
try (MDC.MDCScope scope = MDC.scope("orderId", "ORD-001")) {
    OpenLog.info("处理订单");  // 日志包含 orderId
}
// 离开scope后自动清理

// 多键值作用域
try (MDC.MDCScope scope = MDC.scope(Map.of("traceId", "abc", "userId", "user1"))) {
    OpenLog.info("多上下文");
}
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `put(String key, String value)` | `key` - 键；`value` - 值 | `void` | 设置值 |
| `get(String key)` | `key` - 键 | `String` | 获取值 |
| `remove(String key)` | `key` - 键 | `void` | 移除值 |
| `clear()` | - | `void` | 清空所有 |
| `getCopyOfContextMap()` | - | `Map<String, String>` | 获取上下文副本 |
| `setContextMap(Map<String, String> contextMap)` | `contextMap` - 上下文Map | `void` | 设置上下文Map |
| `runWith(String key, String value, Runnable runnable)` | `key/value` - 上下文；`runnable` - 任务 | `void` | 在指定上下文中执行 |
| `callWith(String key, String value, Supplier<T> supplier)` | `key/value` - 上下文；`supplier` - 任务 | `T` | 在指定上下文中执行（有返回值） |
| `scope(String key, String value)` | `key` - 键；`value` - 值 | `MDCScope` | 创建MDC作用域 |
| `scope(Map<String, String> values)` | `values` - 键值对 | `MDCScope` | 创建多键值MDC作用域 |

### 4.2 NDC - 嵌套诊断上下文

基于栈结构的嵌套上下文，适合跟踪嵌套操作或调用层次。

```java
NDC.push("进入 processOrder");
NDC.push("处理 item-X");
String top = NDC.peek();       // "处理 item-X"
int depth = NDC.getDepth();    // 2
String popped = NDC.pop();     // "处理 item-X"
NDC.clear();

// 使用作用域
try (NDC.NDCScope scope = NDC.scope("处理订单")) {
    // 自动push，离开时自动pop
    OpenLog.info("嵌套日志");
}

// 控制最大深度
NDC.setMaxDepth(10);

// 获取/设置栈副本
Deque<String> stack = NDC.getCopyOfStack();
NDC.setStack(stack);
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `push(String message)` | `message` - 消息 | `void` | 推入栈 |
| `pop()` | - | `String` | 弹出栈顶 |
| `peek()` | - | `String` | 查看栈顶 |
| `clear()` | - | `void` | 清空栈 |
| `getDepth()` | - | `int` | 获取栈深度 |
| `setMaxDepth(int maxDepth)` | `maxDepth` - 最大深度 | `void` | 设置最大深度 |
| `getCopyOfStack()` | - | `Deque<String>` | 获取栈副本 |
| `setStack(Deque<String> stack)` | `stack` - 栈 | `void` | 设置栈 |
| `scope(String message)` | `message` - 消息 | `NDCScope` | 创建NDC作用域 |

### 4.3 LogContext - 统一上下文

整合 MDC 的便捷操作，提供常用上下文字段的快捷方法和上下文快照功能。

**预定义键常量：**

| 常量 | 值 | 说明 |
|------|-----|------|
| `KEY_TRACE_ID` | `"traceId"` | 追踪ID |
| `KEY_REQUEST_ID` | `"requestId"` | 请求ID |
| `KEY_USER_ID` | `"userId"` | 用户ID |
| `KEY_TENANT_ID` | `"tenantId"` | 租户ID |
| `KEY_SPAN_ID` | `"spanId"` | SpanID |
| `KEY_PARENT_SPAN_ID` | `"parentSpanId"` | 父SpanID |

```java
// 设置常用上下文
LogContext.setTraceId("trace-12345");
LogContext.setUserId("user-001");
LogContext.setRequestId("req-abc");
LogContext.setTenantId("tenant-001");

// 获取
String traceId = LogContext.getTraceId();
String userId = LogContext.getUserId();

// 自定义键值
LogContext.set("customKey", "customValue");
String value = LogContext.get("customKey");

// 获取所有上下文 / 清空
Map<String, String> all = LogContext.getAll();
LogContext.clear();

// 上下文快照（用于异步传递）
ContextSnapshot snapshot = LogContext.snapshot();
executor.submit(() -> {
    snapshot.runWith(() -> {
        // 保持原有上下文
        OpenLog.info("异步任务");
    });
});

// 带返回值的上下文传递
String result = snapshot.callWith(() -> {
    return processInContext();
});
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `setTraceId(String traceId)` | `traceId` - 追踪ID | `void` | 设置追踪ID |
| `getTraceId()` | - | `String` | 获取追踪ID |
| `setRequestId(String requestId)` | `requestId` - 请求ID | `void` | 设置请求ID |
| `getRequestId()` | - | `String` | 获取请求ID |
| `setUserId(String userId)` | `userId` - 用户ID | `void` | 设置用户ID |
| `getUserId()` | - | `String` | 获取用户ID |
| `setTenantId(String tenantId)` | `tenantId` - 租户ID | `void` | 设置租户ID |
| `getTenantId()` | - | `String` | 获取租户ID |
| `set(String key, String value)` | `key` - 键；`value` - 值 | `void` | 设置自定义键值 |
| `get(String key)` | `key` - 键 | `String` | 获取自定义键值 |
| `clear()` | - | `void` | 清空上下文 |
| `getAll()` | - | `Map<String, String>` | 获取所有上下文 |
| `snapshot()` | - | `ContextSnapshot` | 创建上下文快照 |
| `apply(ContextSnapshot snapshot)` | `snapshot` - 快照 | `void` | 应用上下文快照 |

**ContextSnapshot 方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `mdcContext()` | - | `Map<String, String>` | 获取MDC上下文副本 |
| `runWith(Runnable runnable)` | `runnable` - 任务 | `void` | 在此上下文中执行 |
| `callWith(Supplier<T> supplier)` | `supplier` - 任务 | `T` | 在此上下文中执行（有返回值） |

### 8.2 LogMasking - 日志脱敏

支持多种脱敏策略，保护敏感数据。

**MaskingStrategy 枚举：**

| 策略 | 效果示例 | 说明 |
|------|---------|------|
| `FULL` | `******` | 全部隐藏 |
| `PHONE` | `138****5678` | 手机号脱敏 |
| `EMAIL` | `t***@example.com` | 邮箱脱敏 |
| `ID_CARD` | `110***********1234` | 身份证脱敏 |
| `BANK_CARD` | `************1234` | 银行卡脱敏 |
| `PASSWORD` | `[PROTECTED]` | 密码脱敏 |
| `NAME` | 部分隐藏 | 姓名脱敏 |
| `ADDRESS` | 部分隐藏 | 地址脱敏 |
| `CUSTOM` | 自定义 | 自定义策略 |

```java
// 直接脱敏
String masked = LogMasking.mask("13812345678", MaskingStrategy.PHONE);  // 138****5678
String masked = LogMasking.mask("test@example.com", MaskingStrategy.EMAIL);  // t***@example.com
String masked = LogMasking.mask("secret123", MaskingStrategy.PASSWORD);  // [PROTECTED]

// 注册脱敏规则
LogMasking.registerRule("password", MaskingStrategy.PASSWORD);
LogMasking.registerRule("phone", MaskingStrategy.PHONE);
LogMasking.registerRule("idCard", MaskingStrategy.ID_CARD);

// 按字段名脱敏
String masked = LogMasking.maskByField("phone", "13812345678");
boolean should = LogMasking.shouldMask("password");  // true

// 注册自定义正则规则
LogMasking.registerPatternRule("creditCard", value -> "****" + value.substring(value.length() - 4));

// 清除规则
LogMasking.clearRules();
```

**`@Mask` 注解：**

```java
public record UserDTO(
    String name,
    @Mask(MaskingStrategy.PHONE) String phone,
    @Mask(MaskingStrategy.PASSWORD) String password
) {}
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `mask(String value, MaskingStrategy strategy)` | `value` - 原值；`strategy` - 策略 | `String` | 脱敏字符串 |
| `maskByField(String fieldName, String value)` | `fieldName` - 字段名；`value` - 原值 | `String` | 按字段名脱敏 |
| `shouldMask(String fieldName)` | `fieldName` - 字段名 | `boolean` | 判断字段是否需要脱敏 |
| `registerRule(String fieldName, MaskingStrategy strategy)` | `fieldName` - 字段名；`strategy` - 策略 | `void` | 注册脱敏规则 |

### 9.1 VirtualThreadContext - Virtual Thread 上下文传播

传统 ThreadLocal 基于 MDC 的上下文不会自动继承到 Virtual Thread。`VirtualThreadContext` 解决此问题。

```java
LogContext.setTraceId("trace-123");
LogContext.setUserId("user-456");

// 启动带上下文的 Virtual Thread
VirtualThreadContext.startVirtualThread(() -> {
    // 自动继承 traceId 和 userId
    OpenLog.info("异步处理");
});

// 创建 Virtual Thread 执行器
ExecutorService executor = VirtualThreadContext.newVirtualThreadExecutor();
ExecutorService executor = VirtualThreadContext.newVirtualThreadExecutor("worker-");
executor.submit(() -> {
    OpenLog.info("线程池任务");  // 上下文自动传播
});

// 包装函数式接口
Runnable wrapped = VirtualThreadContext.wrap(runnable);
Callable<T> wrapped = VirtualThreadContext.wrap(callable);
Function<T, R> wrapped = VirtualThreadContext.wrap(function);
CompletableFuture<T> wrapped = VirtualThreadContext.wrap(future);

// 异步工具
CompletableFuture<Void> f1 = VirtualThreadContext.runAsync(() -> task());
CompletableFuture<T> f2 = VirtualThreadContext.supplyAsync(() -> compute());

// 带上下文直接执行
VirtualThreadContext.runWithContext("trace-123", "user-456", () -> {
    OpenLog.info("带上下文执行");
});
String result = VirtualThreadContext.callWithContext("trace-123", "user-456", () -> {
    return compute();
});
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `startVirtualThread(Runnable task)` | `task` - 任务 | `Thread` | 启动带上下文的Virtual Thread |
| `virtualThreadBuilder(String name)` | `name` - 线程名 | `Thread.Builder.OfVirtual` | 创建Virtual Thread Builder |
| `newVirtualThreadExecutor()` | - | `ExecutorService` | 创建Virtual Thread执行器 |
| `newVirtualThreadExecutor(String namePrefix)` | `namePrefix` - 名称前缀 | `ExecutorService` | 创建带名称前缀的执行器 |
| `wrap(Runnable runnable)` | `runnable` - 任务 | `Runnable` | 包装Runnable传播上下文 |
| `wrap(Callable<T> callable)` | `callable` - 任务 | `Callable<T>` | 包装Callable传播上下文 |
| `wrap(Function<T, R> function)` | `function` - 函数 | `Function<T, R>` | 包装Function传播上下文 |
| `wrap(CompletableFuture<T> future)` | `future` - Future | `CompletableFuture<T>` | 包装Future传播上下文 |
| `runAsync(Runnable task)` | `task` - 任务 | `CompletableFuture<Void>` | 异步执行（Virtual Thread） |
| `supplyAsync(Callable<T> supplier)` | `supplier` - 任务 | `CompletableFuture<T>` | 异步执行（有返回值） |
| `runWithContext(String traceId, String userId, Runnable)` | traceId、userId、任务 | `void` | 带上下文执行 |
| `callWithContext(String traceId, String userId, Callable<T>)` | traceId、userId、任务 | `T` | 带上下文执行（有返回值） |

### 9.2 ScopedLogContext - ScopedValue 日志上下文

基于 JDK 25 ScopedValue 的日志上下文，比 ThreadLocal 更适合 Virtual Thread。

**预定义 ScopedValue：**

| 常量 | 说明 |
|------|------|
| `TRACE_ID` | 追踪ID |
| `USER_ID` | 用户ID |
| `REQUEST_ID` | 请求ID |
| `TENANT_ID` | 租户ID |
| `SESSION_ID` | 会话ID |
| `SPAN_ID` | SpanID |
| `OPERATION` | 操作名 |

```java
// 绑定并执行
ScopedLogContext.where(ScopedLogContext.TRACE_ID, "trace-123")
    .and(ScopedLogContext.USER_ID, "user-456")
    .run(() -> {
        OpenLog.info("处理请求");  // 自动包含 traceId 和 userId

        // 嵌套作用域
        ScopedLogContext.where(ScopedLogContext.REQUEST_ID, "req-789")
            .run(() -> {
                OpenLog.info("嵌套处理");  // 包含所有三个上下文值
            });
    });

// 批量绑定
ScopedLogContext.where(Map.of(
    ScopedLogContext.TRACE_ID, "trace-123",
    ScopedLogContext.USER_ID, "user-456"
)).run(() -> processRequest());

// 快捷方法
ScopedLogContext.run("trace-123", "user-456", () -> task());
String result = ScopedLogContext.call("trace-123", "user-456", () -> compute());

// 获取值
String traceId = ScopedLogContext.getTraceId();
String userId = ScopedLogContext.getUserId();
Optional<String> value = ScopedLogContext.get(ScopedLogContext.OPERATION);
```

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `where(ScopedValue<String> key, String value)` | `key` - ScopedValue；`value` - 值 | `Carrier` | 绑定单个值 |
| `where(Map<ScopedValue<String>, String> bindings)` | `bindings` - 绑定集合 | `Carrier` | 批量绑定 |
| `run(String traceId, String userId, Runnable task)` | traceId、userId、任务 | `void` | 快捷执行 |
| `call(String traceId, String userId, Callable<T> task)` | traceId、userId、任务 | `T` | 快捷执行（有返回值） |
| `getTraceId()` / `getUserId()` / `getRequestId()` / `getTenantId()` / `getSessionId()` / `getSpanId()` / `getOperation()` | - | `String` | 获取各上下文值 |
| `get(ScopedValue<String> key)` | `key` - ScopedValue | `Optional<String>` | 获取自定义值 |

**Carrier 方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `and(ScopedValue<String> key, String value)` | `key` - ScopedValue；`value` - 值 | `Carrier` | 链式绑定 |
| `run(Runnable task)` | `task` - 任务 | `void` | 在绑定上下文中执行 |
| `call(Callable<T> task)` | `task` - 任务 | `T` | 在绑定上下文中执行 |
| `underlying()` | - | `ScopedValue.Carrier` | 获取底层Carrier |

## 13. 线程安全

| 组件 | 线程安全级别 | 实现方式 | 使用建议 |
|------|-------------|---------|---------|
| OpenLog | 完全安全 | 无状态静态方法 | 任意并发调用 |
| Logger | 完全安全 | 不可变对象 | 可缓存共享 |
| LoggerFactory | 完全安全 | 单例缓存 | 全局使用 |
| MDC | 线程隔离 | ThreadLocal | 注意清理 |
| NDC | 线程隔离 | ThreadLocal Stack | push/pop 配对 |
| LogContext | 完全安全 | 委托MDC | 注意清理 |
| ScopedLogContext | 完全安全 | ScopedValues | Virtual Thread 推荐 |
| PerfLog | 完全安全 | 无状态方法 | 任意调用 |
| StopWatch | 非线程安全 | 可变状态 | 单线程使用 |
| AuditLog | 完全安全 | 不可变事件 | 任意调用 |
| LogMasking | 完全安全 | ConcurrentHashMap | 任意调用 |

## 14. 最佳实践

1. **使用占位符而非字符串拼接**：`OpenLog.info("user: {}", user)` 而非 `OpenLog.info("user: " + user)`
2. **Lambda延迟求值**：复杂日志使用 `OpenLog.debug(() -> expensiveOperation())`
3. **缓存Logger**：`private static final Logger log = LoggerFactory.getLogger(MyClass.class)`
4. **合理使用日志级别**：ERROR=异常，WARN=告警，INFO=关键业务，DEBUG=调试
5. **MDC传递上下文**：使用traceId贯穿请求链路，注意finally中清理
6. **使用scope自动清理**：`try (var scope = MDC.scope(...))` 替代手动 put/clear
7. **性能日志独立**：使用PerfLog记录方法耗时
8. **审计日志分离**：敏感操作使用AuditLog单独记录
9. **敏感信息脱敏**：使用 LogMasking 自动脱敏
10. **高频日志采样**：SampledLog 控制日志量
11. **Virtual Thread 传播上下文**：使用 VirtualThreadContext 或 ScopedLogContext
12. **结构化日志便于分析**：使用 StructuredLog 输出 JSON 格式
